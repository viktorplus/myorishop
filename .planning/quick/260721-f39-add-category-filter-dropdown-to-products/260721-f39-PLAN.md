<objective>
Add a category filter to both catalog list pages, per an explicit operator request after today's dictionary-rubric work made «Категория» visible in both places.

**`/products`** already HAS a category filter — but it's a free-text `<input>` doing a substring match against `Product.category` (a free-text field, only ever autofilled once from a Dictionary rubric — see `app/routes/products.py::_products_context`, `app/services/catalog.py::list_products_view`/`category_options`). Typing an exact category name works today, but there's no way to discover the available values without already knowing them. Convert this ONE `<th>` from a text input to a `<select>` sourced from `category_options(session)` (already imported in `app/routes/products.py`, already used to feed the product-form's `<datalist>` — reuse it, don't duplicate it). The underlying filter query param name (`category`) and matching logic in `list_products_view` do NOT change — a select just constrains what gets typed into the same param.

**`/dictionary`** has NO category/rubric filter at all today — the «Категория» column (added earlier today, quick task 260720-wqc) is display-only. `Dictionary.rubric` is drawn from the closed `app.services.rubrics.RUBRICS` list (15 fixed values, CAT-06), so this is a genuinely new filter: a `<select>` sourced directly from `RUBRICS` (not from a DB query — the vocabulary is a Python constant, no need to hit the DB for distinct values), threaded through `GET /dictionary`'s query params exactly like the existing `code`/`name`/`sort` filters (see `app/routes/dictionary.py::_dictionary_context`, `app/services/dictionary.py::list_entries`).

Both pages already have an established `<select>` filter idiom (the sort dropdown on both pages, and the type/author/product filters on `/history` — `partials/history_rows.html`) to copy verbatim: `hx-get` to the list route, `hx-trigger="change"`, `hx-include="#<rows-id> input, #<rows-id> select"`, `hx-target="#<rows-id>"`, `hx-swap="outerHTML"`, `hx-push-url="true"`, with a leading `value=""` "all" option.

Output: both /products and /dictionary have a working, discoverable category filter dropdown.
</objective>
