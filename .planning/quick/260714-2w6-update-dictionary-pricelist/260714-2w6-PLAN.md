  <action>
Create a new standalone script mirroring the conventions in scripts/import_prices.py and scripts/import_catalogs.py (sys.path.insert for the project root, PROJECT_ROOT constant, argparse, SessionLocal from app.db, new_id/to_cents from app.core).

CLI: argparse with a single --file option, default "catalogs/oriflame_prices_with_calculations_fixed.xlsx", resolved relative to PROJECT_ROOT when not absolute (same pattern as import_catalogs.py's --file). Exit with a clear message if the file does not exist.

Load the workbook with openpyxl.load_workbook(path, read_only=True, data_only=True) and select the sheet named "Прайс-лист" (fail loudly if that sheet name is missing — do not silently fall back to another sheet).

Column mapping: read the first row via iter_rows(values_only=True, max_row=1) and build a colmap by matching each header cell (stripped) against the five expected header names Код, Название, ДЦ, ПЦ, Последний каталог. Exit with a clear error naming any missing expected column (defensive against a re-exported file with renamed/reordered columns).

Row parsing (iterate every row after the header): skip rows where the Код cell is None/blank. Code -> str(value).strip() (values come through openpyxl as int for this column). Name -> str(value).strip()[:200] if present else None (matches String(200) columns). ДЦ -> consultant_cents and ПЦ -> consumer_cents: apply the same _cents-style guard as scripts/import_prices.py (must be an int/float > 0, then app.core.to_cents(str(value)); otherwise None — do NOT skip the row for a missing price, since ~26 sampler/tester rows in this sheet legitimately have both prices blank and must still get a Dictionary + CatalogPrice row with null prices).

Catalog parsing: write a helper (e.g. parse_last_catalog(value: str) -> tuple[int, int] | None) that extracts digit groups from the "Последний каталог" cell via re.findall(r"\d+", str(value).strip()) and disambiguates using the exact same heuristic as parse_catalog() in scripts/import_prices.py: with nums = [a, b], if len(nums[0]) == 4 return (a, b) [YYYY-MM]; elif len(nums[1]) == 4 return (b, a) [MM-YYYY] — this is the branch that fires for the sheet's actual "17-2021"-style values, yielding (2021, 17); elif a > 17 return (2000+a, b) [YY-MM]; elif b > 17 return (2000+b, a) [MM-YY]; else return None. Skip (and count as skipped) any row where this returns None or where fewer than 2 digit groups were found.

Deduplicate by code into a dict (last row wins, mirroring import_prices.py's `collected` dict pattern) mapping code -> {name, year, number, consumer_cents, consultant_cents}.

Full replace inside a single `with SessionLocal() as session:` block: capture before_dict = session.query(Dictionary).count() and before_cp = session.query(CatalogPrice).count(); delete all rows from both tables (session.query(Dictionary).delete() and session.query(CatalogPrice).delete()); bulk_save_objects a Dictionary row per collected code (id=new_id(), code=code, name=data["name"], catalogs=[app.services.catalogs.to_json_code(data["year"], data["number"])] — a single-element list, this is the key behavior change from many-catalogs history to just-the-latest-one); bulk_save_objects a CatalogPrice row per collected code (id=new_id(), year=data["year"], number=data["number"], code=code, name=data["name"], consumer_cents=data["consumer_cents"], consultant_cents=data["consultant_cents"], points=None — this sheet has no ББ/points column); commit; then re-query both counts as after_dict/after_cp.

Do NOT touch Product, Batch, Operation, Sale, or any product/ledger table — Dictionary and CatalogPrice are pure helper tables (D-24 in app/models.py docstrings), fully derived and safely reimportable.

Print a summary: source path, sheet name, total data rows scanned, rows imported (len of the deduped dict), rows skipped with a reason breakdown (missing code vs. unparsable catalog value), and "Dictionary: {before_dict} -> {after_dict}" / "CatalogPrice: {before_cp} -> {after_cp}".
