<objective>
An explicit operator request: a safe way to wipe test/business data back to a clean baseline on BOTH the local SQLite install and the deployed PostgreSQL server (same script, dialect-aware), plus a companion script to load a known-quantity test dataset (10 products, 10 customers, 10 operations of every ledger operation type) for manual QA after a reset.

**Why this needs new code, not just the existing `scripts/reset_demo_data.py`/`scripts/seed_demo_data.py` pair:** those two scripts already exist but don't fit this request:
- `reset_demo_data.py` deletes the SQLite **file** and re-migrates — it wipes literally everything (users, dictionary, device tokens included) and only works locally (there is no "delete the file" equivalent for the deployed PostgreSQL server; `deploy/DEPLOY.s1.md`'s only documented server-wipe is `docker compose down -v`, which drops the ENTIRE Postgres volume including all device tokens/users/dictionary — far more destructive than intended here).
- The operator confirmed (mid-conversation) the desired scope is narrower: wipe products/customers/sales/operations/cash/batches, but KEEP users, device tokens, the dictionary/catalog-price reference data, and warehouses. Neither existing script does that.
- `seed_demo_data.py` creates 5 products/3 customers/3 sales — not the "10 products, 10 customers, 10 operations of each type" shape requested, and doesn't cover writeoff/return/correction/transfer/price_change at all.

**The append-only-ledger constraint (CLAUDE.md, `app/models.py` Operation/CashMovement) is the central engineering problem here.** `operations`/`cash_movements` carry `BEFORE UPDATE`/`BEFORE DELETE` triggers that unconditionally `RAISE(ABORT, ...)` on SQLite (`app/db.py::APPEND_ONLY_TRIGGERS`, frozen in Alembic migrations 0001/0013/0018) and an equivalent PL/pgSQL trigger function on PostgreSQL (migration 0001, gated on `dialect.name == "postgresql"`). A bare `DELETE FROM operations` is blocked on BOTH dialects. This is deliberate and must stay true for the live app after this script runs — the reset script may bypass the triggers ONLY for the duration of its own wipe (a documented, deliberate, interactively-confirmed exception), then MUST restore them so the app's normal append-only guarantee is unaffected.

Output: `scripts/reset_business_data.py` (interactive, dialect-aware, typed-confirmation-gated) and `scripts/load_test_data.py` (service-layer, 10 customers + 90 operation rows spanning all 9 types), both testable via an importable core function.
</objective>

Structure:
```python
from sqlalchemy import text
from app.db import SessionLocal, engine
from app.db import APPEND_ONLY_TRIGGERS
from app.models import Batch, CashMovement, Customer, CustomerContact, Operation, Product, Sale

# Children-before-parents delete order (matches the FK graph — see app/models.py).
# warehouses/users/device_tokens/dictionary/catalog_prices/active_catalog/sync_state
# are NEVER touched by this script.
_WIPE_ORDER = (CashMovement, Operation, CustomerContact, Sale, Batch, Customer, Product)

_TRIGGER_NAMES = (
    "operations_no_update", "operations_no_delete",
    "cash_movements_no_update", "cash_movements_no_delete",
)


def wipe_business_data(session, engine) -> dict[str, int]:
    """Delete every row from _WIPE_ORDER's tables; return {table_name: rows_deleted}.

    Bypasses the append-only triggers on operations/cash_movements for the
    duration of this call ONLY, then restores them before returning — the
    live app's UPDATE/DELETE guarantee on those two tables is unaffected
    once this function returns, on EITHER dialect."""
    dialect = engine.dialect.name
    counts: dict[str, int] = {}
    if dialect == "sqlite":
        for name in _TRIGGER_NAMES:
            session.execute(text(f"DROP TRIGGER IF EXISTS {name}"))
    elif dialect == "postgresql":
        session.execute(text("ALTER TABLE operations DISABLE TRIGGER ALL"))
        session.execute(text("ALTER TABLE cash_movements DISABLE TRIGGER ALL"))
    try:
        for model in _WIPE_ORDER:
            result = session.execute(model.__table__.delete())
            counts[model.__tablename__] = result.rowcount
    finally:
        if dialect == "sqlite":
            for ddl in APPEND_ONLY_TRIGGERS:
                session.execute(text(ddl))
        elif dialect == "postgresql":
            session.execute(text("ALTER TABLE operations ENABLE TRIGGER ALL"))
            session.execute(text("ALTER TABLE cash_movements ENABLE TRIGGER ALL"))
    session.commit()
    return counts
```
