<objective>
Build the pure, unit-testable state + presentation layer of the sync client
BEFORE the network driver (Plan 03) that consumes it. Nothing here touches the
network; everything is a plain function over a `Session`.

Delivered:
- `SyncResult` dataclass (status + pushed/pushed_total/pulled counts) — the value
  object the driver returns and the formatter renders.
- `get_or_create_sync_state(session)` + `record_sync_result(session, result)` — the
  single-row `sync_state` (id=1) upsert, written after EVERY attempt from one exit
  point so a failure is recorded as reliably as a success and survives a restart
  (D-10). Portable SELECT-then-INSERT (no `INSERT OR REPLACE`).
- `read_autosync_config(session)` — reads `auto_enabled` + `auto_interval_seconds`
  FRESH from `sync_state` (so flipping the toggle takes effect next tick — D-08/D-15)
  and clamps the interval into 60..3600 seconds.
- `unsynced_count(session)` — the D-11 badge: `COUNT(*) WHERE synced_at IS NULL`
  across `Operation` + `CashMovement`; the caller hides the badge at 0.
- `format_sync_message(result, sync_state, tz)` — the LOCKED D-12 Russian strings,
  with the last-sync line formatted in `settings.display_tz` (Europe/Moscow).

Purpose: isolate the deterministic logic (state persistence, counting, RU copy)
into fast unit tests so Plan 03's network driver only has to wire push/pull.
Output: `app/services/sync_client.py` (state+presentation helpers) + the first
half of `tests/test_sync_client.py`.
</objective>
