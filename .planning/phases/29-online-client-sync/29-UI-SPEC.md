# Phase 29 — UI Design Contract

> Visual and interaction contract for the Online Client Sync surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This is an EXISTING app with a locked design system (28 shipped phases).** No new design tokens are introduced. Every value below is REUSED verbatim from `app/static/style.css` and the established conventions. New work is limited to *interaction contracts* and *Russian copy* for the sync surface; visuals reuse existing classes.

## Spacing Scale

Project scale (multiples of 4), as normalized in the Phase 2 UI-SPEC and used throughout `style.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/label gaps, `label` margin, tight inline padding |
| sm | 8px | `nav` link padding, input padding, compact gaps, badge inset |
| md | 16px | `nav` gap, default element/field spacing, `.container` gutter |
| lg | 24px | Section spacing, `.form-actions` top margin |
| xl | 32px | Layout gaps, `.container` bottom padding |
| 2xl | 48px | `.empty-state` vertical padding |

Exceptions:
- **64px (3xl) is NOT used in this project** — do not introduce it.
- **44px touch-target minimum** exists project-wide for `/m/*` mobile pages only. **Phase 29 is desktop-only (server-only mobile, out of scope)**, so the 44px exception does NOT apply here. The sync button, badge, and settings controls sit in the desktop `<nav>` / Settings page and use the standard `sm/md` scale.

## Typography

Reused verbatim from `style.css` (4 sizes, 2 weights — the locked project system):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / control text | 16px | 400 | 1.5 |
| Label / meta / status caption | 14px | 400 (600 for emphasis) | 1.5 |
| Section heading (`h2`) | 20px | 600 | 1.2 |
| Page heading (`h1`) | 24px | 600 | 1.2 |

Sync-surface application:
- **Sync status line + last-sync line + result message** → 14px muted caption (`.muted`, style.css:272), matching the Settings `<p class="muted">` caption precedent (settings.html:6,10). Rationale: glanceable-but-unobtrusive, never competes with page content.
- **«Синхронизировать» nav link** → inherits the 16px `nav a` treatment (no size change).
- **Unsynced-count badge** → 14px, weight 600 (emphasis on the number).
- **Settings toggle/interval labels** → 14px `label` (style.css:97-101).

## Interaction States (the six required states for the sync surface)

| State | Trigger | Visible result | Colour |
|-------|---------|----------------|--------|
| Idle / never synced | Fresh install, no sync yet | `Ещё не синхронизировано`; badge shows unsynced count if `> 0` | Muted (badge amber) |
| In-flight | Click «Синхронизировать» / auto tick | `Синхронизация…` via hx-indicator; page stays fully interactive | Muted |
| Success | Push+pull OK | D-12 success string + updated last-sync line; badge recomputed (hidden if now 0) | Muted |
| Partial | Some rows pushed | `Синхронизировано частично…` | Amber |
| Offline | `httpx.ConnectError`/timeout caught | `Нет связи с сервером`; **handler returns 200 with OOB partial, never 5xx**; local work unaffected (SRV-03) | Amber |
| Error | Server 5xx/4xx | `Ошибка сервера, попробуйте позже`; still 200 + partial, non-blocking | Destructive |

Two secondary states: **lock-hit** (`Синхронизация уже выполняется`, D-09) and **not-configured** (`Синхронизация не настроена`). Both render as ordinary OOB partials at 200 — no error page, no page reload.

## Accessibility Notes

- **No colour-only meaning** — every state carries its Russian text string (WCAG 1.4.1). Badge conveys a number.
- **Focus** — the trigger reuses the existing `a:focus-visible { outline: 2px solid #2563eb }` ring (style.css:203-209); do not suppress it.
- **Non-blocking** — sync never disables navigation or forms; the in-flight indicator is advisory only (SYNC-06/SRV-03).
- **Autoescaping** — result strings are server-rendered fixed text; the pushed/pulled counts are integers. Never render raw server error bytes or the sync token into the partial (Security Domain; D-12 fixed strings only).
