<objective>
Build `app/services/dashboard.py`, the single new service module composing
everything Главная needs: DASH-01 (date/weekday/time), DASH-02's catalog
countdown (reading Plan 01's `ActiveCatalog` row), DASH-03 (today/week/month
revenue+profit+expense, D-07's expense definition + D-08's net-profit
formula), DASH-04 (distinct-code count + stock valuation), and DASH-05 (a
10-row recent-operations feed spanning all 6 stock-affecting types). Zero
ledger writes — 100% read composition over already-shipped Phase 6/16/17
reporting services. Per D-09, this is achieved by generalizing two existing
single-purpose shapes rather than inventing new aggregation logic:
`_metrics_context`'s single-period composition becomes 3 simultaneous
periods (`period_metrics`/`dashboard_metrics`), and `recent_sales`'s
type-locked feed query becomes a 6-type feed (`recent_operations`); the
distinct-product-code count is the one genuinely new SQL aggregation this
plan adds (nothing existing computes it today).

Purpose: give the desktop (Plan 06) and mobile (Plan 07) home routes ONE
thin call each — `dashboard_context(session, settings.display_tz)` — so
neither route re-derives any of this composition.
Output: `app/services/dashboard.py`, fully tested in isolation (no route/
template work in this plan).
</objective>
