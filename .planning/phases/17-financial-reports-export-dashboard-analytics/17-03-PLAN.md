<objective>
Build the desktop period cash-flow report sub-page `/finance/report` (FIN-08) — a near-verbatim clone of the `/reports/sales` route/template pattern — plus the `/finance/report.csv` download route (FIN-09) that delegates to the 17-01 `stream_cash_movements_csv`. Purpose: Give the operator the heavier detailed movement report + the CSV export on a dedicated sub-page (D-04a), keeping its period control independent from the dashboard tiles (D-04b).
Output: `/finance/report` + `/finance/report.csv` routes, `pages/finance_report.html`, `partials/cash_flow_report.html`, web tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /finance/report & /finance/report.csv (from/to) | Untrusted date params cross here |
| stored note/category → report HTML | Stored category keys resolve to fixed CASH_CATEGORIES labels; no free-text note is rendered in the aggregated report |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | `/finance/report` + `/finance/report.csv` from/to | mitigate | Both flow through `_resolve_period` (clamp malformed/inverted → today + RU error) then `local_day_bounds_utc`; consumed only as ORM `.where` bounds; never string-built, never used as a path/filename |
| T-17-02 | Tampering/Exec | CSV free-text cells | mitigate | Handled in 17-01 `stream_cash_movements_csv` via `_csv_safe`; this route only passes validated bounds |
| T-17-05b | Tampering (XSS) | category label rendered in report HTML | mitigate | Jinja autoescape on; labels come from the server-side `CASH_CATEGORIES` map; NEVER apply `| safe` to any cell (mirrors the T-16-06 rule in cash_history_rows) |
| T-17-06 | DoS | CSV download | mitigate | Period-scoped, streamed (StreamingResponse, not buffered); single-operator scale |
| T-17-09 | Spoofing/Integrity | CSV link behavior | mitigate | Plain `<a href>` download (never `hx-get`) so the browser saves the file rather than swapping CSV into the DOM |
</threat_model>
