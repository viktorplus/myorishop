<objective>
Close the Phase 17 UAT gap (17-UAT.md Test 2, diagnosed in `.planning/debug/finance-report-nav-entry-missing.md`): the CSV export/report page (`/finance/report`, `/m/finance/report`) works correctly but has no prominent, clearly-labeled navigation entry point reachable from the main page. The route/page/CSV logic itself is NOT broken — this is a discoverability/labeling fix only, touching templates.

Root cause (confirmed via full template read, not re-diagnosed here): the report page is reachable only via a bare, unstyled inline `<a>` buried inside a dense dashboard page (`pages/finance.html`, `mobile_pages/finance.html`), or via one link among six crammed into a paragraph on `/reports` (desktop only) — none of the three existing paths uses "export"/"CSV"/"скачать" wording, and no top-level nav item (desktop `<nav>` in base.html, or the mobile home tile grid) points directly at the report page.

Purpose: A tester/operator starting from the main page (desktop `/` or mobile `/m/`) can find and open the CSV export page in one hop, with wording that matches their mental model ("export"/CSV), closing UAT Test 2 (severity: major).
Output: Updated `base.html` (new top nav item), `mobile_pages/home.html` (new mobile tile), relabeled+`.button`-styled links on `pages/finance.html` / `mobile_pages/finance.html`, relabeled link on `pages/reports_landing.html`, and new web tests proving every entry point renders.
</objective>
