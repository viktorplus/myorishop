<objective>
Build the second half of SAL-06: a debounced name-fragment search on the sales page that renders a click-to-select dropdown of matching code+name rows, reusing `search_products()`/`split_match()` verbatim (D-08), and wire it as a SHARED fragment so it survives both the initial basket-row render and every subsequent code-triggered `/sales/lookup` OOB swap.

Purpose: today typing a code on the sales page shows the name inline (already shipped); typing a name fragment does nothing. SAL-06 requires the reverse direction too — a dropdown of matching codes to pick from.
Output: new `GET /sales/search-name` route + response partial; a new shared name-field partial wired into both `sale_row.html` (first render) and `sale_lookup.html` (the existing OOB swap target) so the dropdown never silently disappears.
</objective>
