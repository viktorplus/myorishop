<objective>
Fix two "we already fetched the name and threw it away" bugs found by the full 8-menu audit (D-13 mobile sales, D-14 mobile transfers): both wizards already call the product/dictionary lookup that returns a name, but neither wizard ever displays it after step 1, leaving the operator staring at a bare code through the rest of the flow.

Purpose: SAL-06's "operator sees the product name, not just a code" spirit extends naturally to the mobile sale and transfer wizards — this closes a narrow, already-in-scope gap without pulling forward any of Phase 13's broader mobile navigation/step-indicator work (explicitly out of bounds per the CONTEXT.md boundary note).
Output: mobile sale wizard steps 2-3 and mobile transfer wizard steps 2-3 each show a visible `код — название` line, sourced entirely from data each handler already fetches.
</objective>
