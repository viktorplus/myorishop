<objective>
Build the per-device token identity that the sync endpoints will authenticate with
(SYNC-09): a `device_tokens` table, its migration, and a fat service that mints, verifies,
lists and revokes tokens using stdlib primitives only.

Purpose: a client must be able to prove which device it is without a browser session, and
an administrator must be able to kill a lost device's access instantly.
Output: `DeviceToken` model, migration `0019`, `app/services/devices.py`, unit tests.

Deliberately NOT in this plan: the HTTP dependency (`require_device`), the `auth_guard`
bypass, and the admin UI. This plan is pure model + service so it is unit-testable with the
plain `session` fixture before any app wiring — the Phase 25 Plan 03 precedent.
</objective>

  <action>
Create `app/services/devices.py` following the `app/services/users.py` fat-service contract
exactly (validation lives HERE, never in a route).

Module docstring must state:
- Requirement ID SYNC-09 and that this is the security-critical tier.
- The CLAUDE.md safety line, adapted: the token plaintext is generated, returned to the
  caller ONCE, and is NEVER stored, re-derivable, logged, printed, or echoed into any
  rendered fragment after that single return.
- The DELIBERATE hasher divergence, spelled out so it reads as intentional at review and
  not as an oversight: passwords use Argon2id because they are low-entropy and need a slow
  KDF; a `secrets.token_urlsafe(32)` device token carries 256 bits of CSPRNG entropy, so a
  slow KDF buys nothing against brute force while adding ~50-100 ms of CPU to EVERY sync
  request. SHA-256 is therefore correct here (RESEARCH A1). Reference `28-RESEARCH.md`.

Module constants:
- `TOKEN_PLAINTEXT_PREFIX = "myos_"` — human-recognisable so a leaked string is identifiable
- `TOKEN_ENTROPY_BYTES = 32`
- `TOKEN_PREFIX_LEN = 12`
- RU message constants (HTML-free, matching the `users.py` style): `LABEL_REQUIRED_ERROR = "Укажите название устройства."`, `DEVICE_ID_REQUIRED_ERROR = "Укажите идентификатор устройства."`, `TOKEN_NOT_FOUND_ERROR = "Токен устройства не найден."`

Functions:

`_digest(plaintext: str) -> str` — `hashlib.sha256(plaintext.encode("utf-8")).hexdigest()`.
Private, one line, docstringed.

`mint_token(session, *, device_id, label, user_id=None) -> tuple[tuple[DeviceToken, str] | None, dict[str, str]]`
Validate first with ZERO writes on failure: `label` stripped and non-empty (else
`LABEL_REQUIRED_ERROR` keyed `label`), `device_id` stripped and non-empty (else
`DEVICE_ID_REQUIRED_ERROR` keyed `device_id`), `label` truncated to the `String(100)` width.
On success generate `plaintext = TOKEN_PLAINTEXT_PREFIX + secrets.token_urlsafe(TOKEN_ENTROPY_BYTES)`,
build the row with `token_prefix=plaintext[:TOKEN_PREFIX_LEN]` and `token_hash=_digest(plaintext)`,
`is_active=1`, `session.add` + `session.commit()`, and return `((row, plaintext), {})`.
The plaintext appears in the return value and nowhere else — never assigned to the row,
never logged.

`lookup_active_token(session, presented: str) -> DeviceToken | None`
Return None for a falsy/short `presented` value before touching the DB. Then ONE indexed
`select(DeviceToken).where(DeviceToken.token_prefix == presented[:TOKEN_PREFIX_LEN], DeviceToken.is_active == 1)`
via `session.scalar`. If no row, return None. Otherwise compare `_digest(presented)` against
`row.token_hash` with `compare_token` (imported from `app.services.auth`) and return the row
or None. Never hash every row; never use `==` on the digests. Do not commit here — this is a
read path called on every sync request.

`touch_last_used(session, token: DeviceToken) -> None`
Stamp `last_used_at = utcnow_iso()` and commit. Kept separate from `lookup_active_token` so
verification stays a pure read and the caller decides when to write.

`list_device_tokens(session) -> list[DeviceToken]`
`select(DeviceToken).order_by(DeviceToken.label)` — the admin page feed.

`revoke_token(session, token_id: str) -> tuple[DeviceToken | None, dict[str, str]]`
Soft-disable only, mirroring `users.deactivate_user`: missing row → `(None, {"token": TOKEN_NOT_FOUND_ERROR})`
with zero writes; otherwise set `is_active = 0` and `revoked_at = utcnow_iso()`, commit,
return `(row, {})`. Revoking an already-revoked token is idempotent (re-stamping is fine).
NEVER `session.delete` a DeviceToken row.

Portability: `select()` only, no `sqlalchemy.dialects`, no raw SQL. No `print`, no `logging`
call anywhere in this module.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator/admin → token store | A minted plaintext crosses out of the system exactly once and can never be recovered afterwards |
| database at rest → attacker | A stolen DB dump must not yield usable device credentials |
| presented token → stored secret | Untrusted client-supplied bytes are compared against a stored digest |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | Spoofing | Device-token theft (lost laptop, copied config) | mitigate | Instant admin revocation via `revoke_token` soft-disable (`is_active=0` + `revoked_at`); `last_used_at` stamped by `touch_last_used` makes a stale/unexpected token visible — Tasks 1-3. Residual risk accepted: no expiry, by decision (revocation-only) |
| T-28-07 | Information Disclosure | Token plaintext at rest / in logs | mitigate | Only `hashlib.sha256` hex is persisted; plaintext exists solely in `mint_token`'s return value; zero `print`/`logging` calls in the module (grep-asserted) — Task 2; `test_mint_returns_plaintext_once_and_stores_only_a_hash` proves it — Task 3 |
| T-28-10 | Spoofing | Timing side-channel on token comparison | mitigate | Single indexed `SELECT` on the NON-secret `token_prefix`, then `auth.compare_token` (`hmac.compare_digest`) on the digests; never `==`, never per-row hashing — Task 2; `test_lookup_rejects_a_wrong_token` proves the prefix alone does not authenticate — Task 3 |
| T-28-17 | Spoofing | Weak token generation | mitigate | `secrets.token_urlsafe(32)` = 256 bits CSPRNG; never `random`, never `uuid4` — Task 2; distinctness asserted in `test_two_mints_produce_distinct_tokens` |
| T-28-18 | Repudiation | Losing the audit trail by hard-deleting a compromised token | mitigate | `revoke_token` never calls `session.delete` (grep-asserted); the row and its `device_id`/`user_id` survive revocation — Tasks 2-3 |
