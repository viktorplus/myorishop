Deliver the stock-correction vertical slice (OPS-03) end to end and retire the
walking-skeleton correction path. The operator opens `/corrections`, types a code
(name + current cached quantity autofill), chooses a mode, and saves — recording a
`correction` op through the single write path, never a direct edit of
products.quantity.

stock, system computes qty_delta = counted − current, with the current quantity shown
beside the input; and delta: enter ± directly, written as-is), D-10 (always a
`correction` op via record_operation; a zero net delta is rejected gracefully with no
row), D-11 (optional note; payload {"note","mode"}; default UI mode = counted, the
safer choice), D-12 (this REPLACES the walking-skeleton POST /ops — delete
app/routes/ops.py, remove it from app/main.py, and update the home page form; do NOT
leave two correction paths).

Makes tests/test_corrections.py GREEN (including `ops_replaced`).

Purpose: correct stock counts safely as ledger operations, not silent edits.
Output: corrections service, /corrections routes, correction form, ops.py retired.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /corrections | Untrusted operator input (code, mode, value, note) crosses into the service. |
| correction op → products.quantity | Stock changes ONLY through record_operation; the route never edits products.quantity. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Tampering | POST /corrections mode | mitigate | Server-side allow-list `mode ∈ {"count","delta"}` in register_correction; the radio group is not trusted (V5). |
| T-05-13 | Tampering | correction value parse (signed delta) | mitigate | isascii()+isdigit() guard with explicit single-leading-'-' handling for delta; count mode ≥ 0; garbage → RU 422, never an uncaught ValueError. |
| T-05-14 | Repudiation/Tampering | stock adjustment bypassing the ledger | mitigate | Correction is written ONLY via record_operation (single write path); products.quantity is never edited directly (OPS-03/D-10); append-only triggers ABORT UPDATE/DELETE. |
| T-05-15 | Tampering | duplicate correction path | mitigate | Delete POST /ops and remove it from main.py + home (D-12) — exactly one correction endpoint remains; `test_web_ops_replaced` enforces it. |
| T-05-16 | Info disclosure | POST /corrections exceptions | mitigate | Route try/except → logger.exception + RU 422 partial; never a raw 500 (V7). |
| T-05-17 | Tampering | zero-net / no-op rows | mitigate | `qty_delta == 0` rejected before any write — no meaningless correction rows (D-10). |
| T-05-SC | Tampering | pip installs | accept | No package installs this phase (internal wiring only) — 05-RESEARCH § Package Legitimacy Audit: N/A. |
</threat_model>
