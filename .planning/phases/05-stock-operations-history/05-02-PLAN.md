<objective>
Deliver the write-off vertical slice (OPS-01) end to end: an operator opens
`/writeoff`, types a product code (name autofills via the receipt 204 lookup),
enters a quantity and a required reason category (+ optional note), and saves —
producing one immutable `writeoff` op with qty_delta<0 through the single write
path, decreasing stock.

Honors: D-01 (hybrid reason = required category + optional note), D-02 (payload
`{"reason_code","note"}`; latin code stored, RU label rendered from
WRITEOFF_REASONS), D-03 (the 6-category set), D-04 (by product code, positive-int
qty, NO price fields, stock may go to/through zero with an oversell warn-but-allow
reusing the Phase 4 SAL-04 pattern). Reason capture is designed for Phase 6
RPT-03 grouping.

The slice mirrors the shipped receipt flow almost verbatim (save-and-next,
focus-back-to-Код, 204 autofill, oob recent list) and the sale oversell
warn/confirm. Makes tests/test_writeoffs.py GREEN.

Purpose: the first operator-facing capability of Phase 5.
Output: writeoffs service, /writeoff routes, write-off templates, main.py wiring.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /writeoff | Untrusted operator form input (code, qty, reason_code, note, confirm) crosses into the service. |
| browser → GET /writeoff/lookup | Untrusted code/name query params. |
| stored payload.note → /history render | Operator free-text note is later rendered in the history table (untrusted-at-rest). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | POST /writeoff reason_code | mitigate | Server-side allow-list `reason_code ∈ WRITEOFF_REASONS` in register_writeoff — the `<select>` alone is not trusted (V5). |
| T-05-02 | Tampering | POST /writeoff qty | mitigate | `qty_text.isascii() and qty_text.isdigit()` guard + qty>0; non-ASCII/garbage falls to the RU 422 path, never an uncaught ValueError. |
| T-05-03 | Tampering | write-off driving stock below zero | mitigate | Server-side stock check before write; warn-but-allow with explicit confirm=1 (D-04) — a below-zero write requires a deliberate second POST, never a silent one. |
| T-05-04 | Info disclosure | POST /writeoff exceptions | mitigate | Route try/except → `logger.exception` + RU 422 partial; never a raw 500 stack trace (V7). |
| T-05-05 | Tampering | payload.note / product name in recent-writeoffs render | mitigate | Jinja autoescape ON; never `|safe` on note or product name (stored-XSS). |
| T-05-06 | Tampering | operations UPDATE/DELETE | mitigate | write-off is a NEW row via record_operation; append-only triggers ABORT mutation (inherited invariant, covered by test_ledger append_only). |
| T-05-SC | Tampering | pip installs | accept | No package installs this phase (internal wiring only) — 05-RESEARCH § Package Legitimacy Audit: N/A. |
</threat_model>
