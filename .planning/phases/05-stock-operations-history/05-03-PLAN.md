<objective>
Deliver the sale-linked return vertical slice (OPS-02) end to end: from a specific
`sale` line in the recent-sales list or a customer's purchase history, the operator
clicks «Вернуть», sees a compact return form (context line + returnable hint + qty),
and submits — producing one immutable `return` op with qty_delta>0, linked to the
original sale, increasing stock.

Honors: D-05 (entry point is a specific origin sale op — a recent-sales /
purchase-history row that already exposes op.sale_id, op.product_id, and the frozen
unit prices), D-06 (return op qty_delta>0, sale_id + product_id carried), D-07
(price/cost symmetry — copy the origin op's frozen unit_price_cents/unit_cost_cents,
NEVER the current product card; preserves SAL-05 profit correctness), D-08 (partial
returns; returnable = sold − already-returned aggregated per sale_id+product_id;
over-return rejected before any write).

New logic: the returnable-qty aggregation. Everything else copies the sale freeze +
customer purchase-history aggregate query. Makes tests/test_returns.py GREEN.

Purpose: make sold stock recoverable without corrupting frozen profit figures.
Output: returns service, /returns routes, return form, «Вернуть» entry-point links.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → GET/POST /returns | Untrusted origin_op_id, sale_id, product_id, qty from the operator/DOM. |
| origin sale op → return op | The return trusts ONLY the server-fetched origin op's frozen amounts, not any client-supplied price. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | return price/cost source | mitigate | Copy unit_price_cents/unit_cost_cents from the server-fetched origin `sale` op (D-07); never read Product.*_cents and never accept a client price — preserves SAL-05 profit correctness. |
| T-05-08 | Tampering | forged origin_op_id / sale_id / product_id | mitigate | `session.get(Operation, origin_op_id)`; reject when missing, not type=="sale", or sale_id is None. product_id/sale_id taken from the fetched op, not trusted from the client. |
| T-05-09 | Tampering | over-return inflating stock | mitigate | `returnable_qty` cap (sold − already-returned per sale_id+product_id) enforced before any write (D-08); qty>remaining → RU 422, 0 writes. |
| T-05-10 | Info disclosure | return of a soft-deleted product | mitigate | record_operation raises ValueError (IN-01 guard); route/service catch it → RU 4xx, never a raw 500 (Pitfall 7). |
| T-05-11 | Tampering | operations UPDATE/DELETE | mitigate | Return is a NEW row; append-only triggers ABORT mutation (inherited; covered by test_ledger append_only). |
| T-05-SC | Tampering | pip installs | accept | No package installs this phase (internal wiring only) — 05-RESEARCH § Package Legitimacy Audit: N/A. |
</threat_model>
