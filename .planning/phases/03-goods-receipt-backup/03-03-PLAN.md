**As a** shop operator, **I want to** register goods receipts by product code — name auto-filled, prices captured, every receipt an immutable ledger entry — while the database backs itself up automatically with a proven restore path, **so that** stock on the shelf is always correct and a day of data entry can never be lost.

| ID | Locked decision | This plan |
|----|-----------------|-----------|
| D-08 | VACUUM INTO 'backups/myorishop-YYYYMMDD-HHMMSS.db' — WAL-safe compact standalone copy | implements |
| D-09 | Automatic backup on app startup (before serving, skipped if DB missing/empty) + manual «Backup now» button on /backup | implements |
| D-10 | Keep the most recent 30 backups; older deleted after a successful new backup | implements (settings.backup_keep = 30) |
| D-11 | Documented restore procedure + restore.bat; restore verified at least once via automated test | implements |

**Plan decision PD-11 (filename collisions + ordering):** two backups within the same second would collide on the D-08 name — target-must-not-exist makes VACUUM INTO fail (RESEARCH Pitfall 4). create_backup appends a "-2", "-3"… suffix before ".db" when the target exists. prune_backups and list_backups order by file mtime, not by name, so suffixed files sort correctly.

**Plan decision PD-12 (engine seam for the route):** POST /backup obtains the engine via session.get_bind() — under the test client that is the tmp-path test engine (no test ever VACUUMs the real data/myorishop.db), in production it is the real app engine. create_backup still opens its OWN connection with AUTOCOMMIT isolation (RESEARCH Pitfall 2).

**Plan decision PD-13 (gate placement + run.bat):** the D-09 skip conditions (flag, DB missing, DB empty) live INSIDE startup_backup(), and main.py calls it module-qualified (backup_service.startup_backup()) so tests monkeypatch one seam. RESEARCH Open Question 3's optional pre-migration backup in run.bat is intentionally NOT added: D-09 is satisfied by lifespan, this phase ships no migration, and a second backup per start would churn D-10 retention (halving the effective history window).
