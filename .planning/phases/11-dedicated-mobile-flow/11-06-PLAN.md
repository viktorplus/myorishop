## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Browser → `POST /m/corrections/step/batch-pick` | Untrusted client-supplied `batch_id` must be re-validated against the looked-up product |
| Browser → `POST /m/corrections` | Untrusted `mode` value reaches `register_correction` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-11-16 | Tampering / Elevation of Privilege | `/m/corrections/step/batch-pick` posted `batch_id` | mitigate | Re-validate `candidate.product_id == product.id` (T-09-08 precedent) |
| T-11-17 | Tampering | `mode` posted value | mitigate | `register_correction`'s existing `mode not in ("count", "delta")` server-side allow-list check is unchanged — this plan's radio-row rendering never widens the accepted value set |
| T-11-18 | Repudiation | Over-removal bypass via a client that never re-POSTs `confirm=1` | mitigate | Zero Operation rows are written until `register_correction` returns without `oversell` set |
| T-11-CSRF | Tampering | CSRF | accept | Unchanged app-wide posture |

</threat_model>
