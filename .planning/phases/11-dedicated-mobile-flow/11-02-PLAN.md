<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Browser вЖТ `GET /m/search?q=` | Untrusted free-text query string reaches the database via `search_view`/`search_products` |
| Browser вЖТ `GET /m/search/product/{product_id}` | Untrusted path parameter used as a primary-key lookup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-11-04 | Tampering | `app/routes/mobile_search.py` `q` query param | mitigate | Reuses `app.services.catalog.search_view`/`search_products` unchanged, which already applies `_escape_like` to `%`/`_`/`\` and folds Cyrillic in Python (not SQL `lower()`) вАФ no new LIKE clause is written by this plan |
| T-11-05 | Information Disclosure | `/m/search/product/{id}` per-warehouse batch summary | accept | Same product/stock data already visible on the desktop `/products` page and `/reports/expiry`; single local operator, no new exposure surface |
| T-11-06 | Tampering | `product_id` path parameter (IDOR-shaped) | mitigate | `session.get(Product, product_id)` returns `None` for a nonexistent id and the route responds 404 rather than leaking a stack trace or a different product's data by coincidence; no write path exists on this route |
| T-11-CSRF | Tampering | CSRF | accept | Unchanged app-wide posture вАФ no auth/session exists to ride, consistent with every other route in this app |

</threat_model>
