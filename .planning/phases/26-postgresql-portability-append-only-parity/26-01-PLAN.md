<objective>
Add the PostgreSQL driver, establish `settings.database_url` as the single DB-URL
source of truth, and lay down the failing PG-parity test scaffold (Wave 0) that
Plans 02–03 turn green in CI.

Purpose: SRV-01/SRV-02 both require the same app + migration history to target
PostgreSQL. That is only possible once (a) the psycopg driver is installed and
(b) exactly ONE config value chooses the DB URL for BOTH `alembic/env.py` and
`app/db.py` (RESEARCH Open Question 2). This plan defines that contract and the
proof harness before any dialect-gating code is written (interface-first).

Output: `psycopg[binary]` dependency, `settings.database_url`, and
`tests/test_pg_parity.py` (5 tests, skipped on SQLite, RED in CI until Plans
02–03 land).

Scope fence (this whole phase): NO sync logic. This plan only adds config + a
dependency + a test scaffold.
</objective>

    - app/config.py (full — `db_path` field line 22, `secret_key`/`device_id`
      "env-value-untouched, only default filled" idiom, and the
      `_resolve_local_identity` `@model_validator(mode="after")` lines 45-59)
    - 26-RESEARCH.md §"Standard Stack" (psycopg v3 3.3.4, `[binary]` extra) and

    - 26-PATTERNS.md §`pyproject.toml` and §`app/config.py`
  </read_first>
  <action>
    In pyproject.toml add `"psycopg[binary]==3.3.*"` to `[project].dependencies`,
    preserving the existing minor-pin style (alphabetical placement, e.g. after
    `jinja2`). Then run `uv add "psycopg[binary]"` so uv.lock is synced (version
    3.3.4 verified in RESEARCH; the `[binary]` extra ships prebuilt libpq wheels so
    CI/dev needs no C toolchain).

    In app/config.py add a new field `database_url: str = ""` on `Settings`. The
    empty default lets an explicit `.env`/environment `DATABASE_URL` win via
    pydantic-settings' automatic field→env mapping (field `database_url` ⇒ env var
    `DATABASE_URL`). Inside the existing `_resolve_local_identity`
    `@model_validator(mode="after")`, add: only when `self.database_url` is empty,
    set `self.database_url = f"sqlite:///{self.db_path}"`. This mirrors the
    "env value left untouched; only the empty default replaced" idiom already used
    for `secret_key`/`device_id`. Do NOT hardcode any host/user/password —
    a PostgreSQL URL only ever arrives via env/.env. Keep the field name
    `database_url` exactly so the `DATABASE_URL` env mapping holds. This one value
    is the single source of truth read by `alembic/env.py` and `app/db.py`
    in Plan 03.
  </action>
  <acceptance_criteria>
    - `grep -c "psycopg\[binary\]==3.3" pyproject.toml` returns 1.
    - `uv run python -c "from app.config import settings; print(settings.database_url)"`
      prints `sqlite:///data/myorishop.db` (no DATABASE_URL set).
    - `DATABASE_URL=postgresql+psycopg://u:p@h/db uv run python -c "from app.config import settings; assert settings.database_url=='postgresql+psycopg://u:p@h/db'"`
      exits 0 (env override wins).
    - No literal password/credential string added to config.py or pyproject.toml
      (source assertion: the only URL literal is the `sqlite:///` default).
    - `uv run ruff check app/config.py` passes.
  </acceptance_criteria>
  <done>psycopg[binary]==3.3.* is a project dependency (uv.lock synced); settings.database_url defaults to the sqlite URL and is overridable via DATABASE_URL; no credential hardcoded.</done>
