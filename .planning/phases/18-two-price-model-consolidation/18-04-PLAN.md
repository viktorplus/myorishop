<objective>
Drop `Product.catalog_cents` — remove the ORM attribute AND the database column — now that plans 18-02 and 18-03 have removed every read/write of it. This is the "remove the definition" half of the drop and is deliberately sequenced AFTER the reference removal so no wave leaves `getattr(product, "catalog_cents")` running against a model whose attribute (or column) is gone (the migration-ordering constraint; Pitfall 4).

D-01 makes the 6 live values a deliberate, irreversible discard (there are 0 backfill candidates — verified). D-03 mandates a NATIVE `op.drop_column`, NOT `batch_alter_table`, mirroring the verbatim precedent at `alembic/versions/0002_catalog_dictionary.py:75` — a batch rebuild of `products` would recreate the partial unique index `uq_products_code_active` and risk losing its `sqlite_where` predicate (house rule frozen in `0001:11-14` / `0008:11-17`). D-24 requires a fresh backup snapshot before the drop; the live-DB apply is human-gated because it is irreversible.

Purpose: PROD-05 — the schema now stores exactly two prices (plus the exempt `min_sale_cents` guardrail).
Output: `alembic/versions/0014_drop_product_catalog_cents.py`, `Product` without `catalog_cents`, the migration-reflection test inverted, and an audited pre-drop snapshot.
</objective>
