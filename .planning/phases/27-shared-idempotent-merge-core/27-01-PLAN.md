<objective>
Define the ONE shared exchange format and its pure parse/serialize functions — the
SYNC-04 wire contract that Phase 28 (online sync) and Phase 30 (offline upload)
both reuse without a second implementation. This plan is interface-first: it lands
the dataclasses, format constants, and the kind→model map that Plans 02–04 build
`apply_merge` on top of, plus the NDJSON factory the whole test suite needs.

Purpose: The milestone's correctness engine needs a single, streamable, verbatim
wire schema before any merge logic exists. NDJSON (one JSON object per line, one
header line first, per-line `kind` discriminator) lets a huge upload parse
line-by-line and lets Phase 30's browser file re-emit it with per-line JSON. Getting
parse/serialize verbatim-faithful and strict about malformed input is the foundation
every later guarantee stands on.

Output: `app/services/merge.py` with `FORMAT_VERSION`, `RECORD_KINDS`,
`KIND_TO_MODEL`, the `ExchangeRecord`/`ExchangeBatch`/`Conflict`/`MergeReport`
dataclasses, `parse_exchange`, and `serialize_exchange`; and `tests/test_merge.py`
with the NDJSON batch factory + round-trip / format-rejection tests. NO `apply_merge`
yet (Plan 02), NO reference-conflict logic (Plan 03), NO DB writes at all in this plan.

Design decisions realized here (from 27-RESEARCH.md — no 27-CONTEXT.md exists; these
are the researcher's recommended defaults, made explicit for later sign-off):
- DD-4 (Format): NDJSON, per-line `kind` + one header envelope line first; header
  carries `format_version` (engine compatibility) + `schema_version` (Alembic head,
  for OFF-07 later). (SYNC-04, RESEARCH §"NDJSON Exchange Format" / A1)
- DD-6 (Verbatim carriage): every ledger record carries origin
  id/device_id/seq/author_id/created_by/created_at unchanged; money stays integer
  cents; timestamps stay ISO-8601 UTC text; `synced_at` is emitted null and never
  read from the wire. (SYNC-02/SYNC-04, RESEARCH §"Rules" / A9)
</objective>

    Implement `parse_exchange(lines: Iterable[str]) -> ExchangeBatch` (pure):
    - Iterate lines; skip blank/whitespace-only lines. `json.loads` each remaining line;
      a non-object or a JSONDecodeError raises `ValueError` (RU-not-required: internal
      English message, e.g. `f"malformed NDJSON line: {i}"`).
    - The FIRST non-blank line MUST be `{"kind":"header", ...}` carrying `format_version`;
      if `format_version != FORMAT_VERSION` raise `ValueError` (feeds OFF-07 later). Capture
      `schema_version` and `source_device_id` (both may be absent → default None / "" per
      the dataclass). A missing/duplicate header raises ValueError.
    - Each subsequent line's `kind` MUST be in `RECORD_KINDS` minus "header" (an unknown or
      second "header" kind raises ValueError). Build `ExchangeRecord(kind, data)` where
      `data` is the parsed dict. Validate required identity: every non-header record has a
      non-empty string `id`; every `operation`/`cash_movement` also has `device_id`,
      `seq` (int), `created_at`, `created_by`. Coerce/verify money-ish fields present in the
      record are `int` or null (never float) — a float money value raises ValueError
      (type-confusion guard, V5). Force `synced_at` to None if present (server-owned, never
      trusted from the wire).
    - Return `ExchangeBatch(format_version, schema_version, source_device_id, records)`.
      Do NOT touch a Session, a file, or the network.

    Implement `serialize_exchange` (pure) yielding NDJSON lines, header FIRST. Accept the
    record list plus the envelope fields needed for the header (schema_version,
    source_device_id, generated_at) — e.g.
    `serialize_exchange(records, *, schema_version, source_device_id, generated_at) -> Iterator[str]`.
    Emit line 1 = the header dict (`kind`=="header", `format_version`==FORMAT_VERSION, the
    envelope fields, and a `counts` dict of records-per-kind). Then `json.dumps` one line per
    record (`{"kind": rec.kind, **rec.data}` shape matching the parse input), `ensure_ascii=False`
    so Cyrillic stays readable. The contract is round-trip identity: for any record list `R`,
    parsing the serialized output yields `ExchangeBatch.records == R` (field-for-field, verbatim).

    Add tests to tests/test_merge.py:
    - `test_round_trip`: build records for several kinds (incl. an `operation` and a
      `cash_movement` with explicit origin id/device_id/seq/author_id/created_by), serialize,

    - `test_malformed_line_rejected`: a non-JSON / non-object line raises ValueError.
    - `test_unknown_kind_rejected`: a line with `kind:"bogus"` raises ValueError.
    - `test_missing_header_rejected`: a batch whose first line is a record (no header) raises.
    - `test_synced_at_not_trusted`: a ledger line carrying `synced_at:"…"` parses with
      `data["synced_at"] is None`.
  </action>

## Trust Boundaries

|----------|-------------|

| engine ↔ transport | The engine is pure (no HTTP/file); auth/size caps live at the Phase 28/30 caller (noted, out of scope here). |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-27-03 | Tampering / DoS | `parse_exchange` (malformed / type-confused / unknown-kind / bad-version NDJSON) | mitigate | Strict per-line `json.loads` + schema validation: reject non-object lines, unsupported `format_version`, unknown `kind`, missing header, and float money values — all raise ValueError before any DB touch (ASVS V5). Payload SIZE caps are a transport concern (Phase 28/30) — noted, not owned here. |
| T-27-06 | Integrity | wire `synced_at` field | mitigate | `synced_at` is forced to None on parse — the server-owned sync cursor is never trusted from the wire (relaxation is Phase 28's job). |
| T-27-05 | Tampering | record field values | mitigate | Fields are carried as plain data only; no field is ever `eval`'d or f-stringed into SQL (this plan does no SQL at all; the discipline is enforced in Plans 02–04). |
| T-27-SC | Tampering | package installs | accept | This phase installs NO packages (stdlib `json`/`dataclasses` + already-installed SQLAlchemy). No registry/supply-chain surface; no `T-27-SC` checkpoint needed. |
