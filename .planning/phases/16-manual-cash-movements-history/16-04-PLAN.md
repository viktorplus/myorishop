<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile browser form → POST /m/finance/withdraw|/deposit | Untrusted amount/category/note/confirm; the mobile route delegates ALL validation to `record_manual_movement` (Plan 02) — routes never write cash (D-00c). |
| stored `note` / label → mobile card HTML | Operator-authored `note` rendered into `.mobile-card`; must rely on Jinja autoescape. |
| mobile query `bucket`/`page` → GET history | Untrusted; bucket flows through the service's `CASH_BUCKETS.get`, page clamped server-side. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-06 | Tampering (stored XSS) | `note` / label in `cash_history_cards.html` | mitigate | Autoescape only — `{{ mv.note }}` / `{{ CASH_CATEGORIES.get(...) }}`, NEVER `|safe`. Task 2 asserts a `<script>` note renders escaped; grep asserts no `|safe`. |
| T-16-05 | Tampering | mobile negative-balance warn | mitigate | Warn returns 200 (mobile_base.html also carries the `"422":{"swap":true}` config — Pitfall 6); the `confirm=="1"` re-POST re-runs the server-side gate in `record_manual_movement` (balance recomputed live). |
| T-16-01 / T-16-02 / T-16-03 / T-16-04 | Tampering / Elevation | amount / category / sign / comment | mitigate (upstream) | Enforced in `record_manual_movement` (Plan 02); mobile routes surface the 422 errors, never bypass them. |

| T-16-SC | Tampering | supply chain (pip installs) | accept (N/A) | Zero packages installed. |
| V2/V3/V4 auth/session/access | — | — | accept (N/A) | Single local operator, no auth in v1 (CLAUDE.md). |
