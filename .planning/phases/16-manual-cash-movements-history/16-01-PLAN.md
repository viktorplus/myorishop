<objective>
Extend the cash-ledger category vocabulary so the Phase 16 manual write path and
history view have their source-of-truth constants: add the 7 manual keys to
`CASH_CATEGORIES` (5 withdrawal + 2 deposit, with RU labels per D-01b), add a
`CASH_BUCKETS` category→bucket grouping map and a `CASH_BUCKET_LABELS` bucket→RU
map (D-01a), and register `CASH_CATEGORIES` + `CASH_BUCKET_LABELS` as Jinja
globals (Pitfall 2 fix — only `WRITEOFF_REASONS`/`OPERATION_TYPE_LABELS` are
globals today).

Purpose: The `record_cash_movement` allow-list guard rejects any category not in
`CASH_CATEGORIES`, so the manual keys MUST land before the service/routes that
write them (Pitfall 1). Templates blank-render or raise `UndefinedError` unless
the dict/bucket maps are globals (Pitfall 2). This is Wave 1 — everything
downstream depends on it.
Output: Extended `CASH_CATEGORIES`, `CASH_BUCKETS`, `CASH_BUCKET_LABELS` in
`app/models.py`; two new Jinja globals in `app/routes/__init__.py`; constant
contract tests in `tests/test_finance.py`.
</objective>
