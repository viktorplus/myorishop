<objective>
Extend `list_warehouses()` in `app/services/warehouses.py` with two new page-wide grouped
aggregate queries — distinct-product item count (D-03) and last goods-receipt date (D-04) —
so WH-01's warehouse list can show both metrics without any per-row query.

Purpose: WH-01 requires "Warehouse list shows the current item count and the date of the last
goods receipt for each warehouse." This is the data layer only; the route/template consumers
land in later plans (20-02/20-03).
Output: `list_warehouses()` return dict's `"warehouses"` rows each carry `.item_count` (int,
default 0) and `.last_receipt` (ISO string or None) as plain Python attributes set after the
existing `paginate()` call.
</objective>

    - Test 2 (`test_list_warehouses_last_receipt_none_when_never_received`): a freshly added

    - Test 3 (`test_list_warehouses_last_receipt_ignores_transfer_only_stock`): a warehouse
      whose ONLY stock arrived via a `transfer`-type Operation (i.e. it is a transfer's
      destination warehouse, never received directly) -> `.last_receipt is None` — WH-01 asks

      load-bearing and must be asserted, not assumed.
  </behavior>
  <action>
    In the same post-`paginate()` block from Task 1, add a second grouped query:
    `select(Batch.warehouse_id, func.max(Operation.created_at)).outerjoin(Operation,
    (Operation.batch_id == Batch.id) & (Operation.type == "receipt")).where(
    Batch.warehouse_id.in_(warehouse_ids)).group_by(Batch.warehouse_id)` — use `outerjoin`, NOT
    `.join()` (a plain inner join would silently drop a warehouse whose batches have zero
    receipt-type operations, e.g. a transfer-only destination). Build a `dict(rows)` keyed by

    `w.last_receipt = last_receipts.get(w.id)` (defaults to `None` via `dict.get`, correct for

    ops"). Import `Operation` into the existing `from app.models import ...` line.
