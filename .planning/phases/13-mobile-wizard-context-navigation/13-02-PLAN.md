<objective>
Migrate the mobile write-off wizard from its current full-page-per-step architecture (each step
is its own `{% extends "mobile_base.html" %}` template returned from a plain
`<form method="post">`, "Назад" implemented via `onclick="history.back()"`) to the persistent-shell
+ htmx-fragment-swap architecture every other mobile wizard already uses — mirroring
`mobile_pages/receipts.html` verbatim (D-04). Add the visible code/name/warehouse header (D-01/D-02/
D-03) to all 3 intermediate steps along the way, reusing the `_wizard_header.html` partial Plan
13-01 created.

Purpose: Phase 13 Success Criteria #1 and #2 — write-off is the one wizard whose "Назад" does not
work via `hx-get`/`hx-post` today; this plan closes that gap and the visible-context gap together
since both require touching the same 3 step templates + the shell page + the route file.
Output: `mobile_pages/writeoff.html` restructured as a persistent shell; a new
`writeoff_step_product.html` fragment; all 3 intermediate step templates converted from
full pages to bare fragments with header includes and hx-post "Назад" buttons;
`app/routes/mobile_writeoff.py` serving step 1 as page-or-fragment and threading `name`/
`warehouse_name` end to end.
</objective>
