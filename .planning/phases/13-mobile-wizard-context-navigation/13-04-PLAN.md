<objective>
Fix the transfers wizard's step 2 ("Партия") "Назад" control. 13-CONTEXT.md's D-06 named transfers
as one of the wizards whose back-navigation is "already correct — no changes needed there beyond
D-01/D-02." Direct verification during planning showed this claim does not hold at every step:

open_question_resolution (logged in Plan 13-01) explicitly closed for corrections and receipts.

hx-get/hx-post pattern with no carve-outs; since this plain link fails that literal test, this plan
extends the identical fix to transfers for full consistency, using the same `hx-get` + fragment
technique. (Transfers' step 3 "Назад," `transfers_step_dest.html`, already correctly uses
`hx-post`/`hx-vals` — untouched by this plan.)

Purpose: Phase 13 Success Criterion #2 — closing the one remaining plain-link "Назад" this
phase's own audit surfaced beyond 13-CONTEXT.md's originally-scoped file list.

refactored around it, `mobile_transfers.py`'s `transfers_step_product` serving step 1 as
page-or-fragment with a `code` pre-fill, and `transfers_step_batch.html`'s "Назад" converted to
`hx-get`.
