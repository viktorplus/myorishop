<objective>
Close the single remaining Phase 13 verification gap (13-VERIFICATION.md, Success Criterion #1,
partial): the mobile SALE wizard is the only one of the five mobile wizards that never shows a
`Склад:` (warehouse) line anywhere, even though `open_batches()` deliberately reads batches across
ALL warehouses for the sale flow (`app/services/batches.py` comment: "Sale picker reads all
warehouses"). An operator can pick a batch during a sale without ever learning which warehouse it
came from, before or after picking. Receipt, write-off, correction, and transfer wizards were all
correctly fixed for this in Phase 13 Waves 1/2 — this plan applies the identical pattern to the
sale wizard, the one wizard 13-CONTEXT.md's D-03 implicitly (and incorrectly) treated as already
compliant.

Because sale's batch-pick step can show batches spanning MULTIPLE warehouses at once (unlike the
other four wizards, which resolve a single warehouse early), this needs a PER-CARD warehouse label
on the batch-pick step (via the shared `batch_card_picker.html` partial, extended with an optional
`warehouse_names` param — mirrors `transfers_step_batch.html`'s own already-shipped per-card
`Склад:` line, since transfers' `open_batches()` call is also unfiltered by warehouse). Once a
batch is actually picked (qty/price step, basket review), a single `Склад: {{ warehouse_name }}`
line — identical wording to every other wizard — is enough.

Purpose: Phase 13 Success Criterion #1 ("Every intermediate step of the sale, receipt, write-off,
correction, and transfer mobile wizards displays the product code, name, and warehouse as visible
on-screen text"), requirement UI-02 (currently only partially satisfied because of this exact gap).
Output: `_warehouse_names()` helper in `mobile_sales.py`; `warehouse_names`/`warehouse_name` threaded
through 4 call sites; a per-card `Склад:` line in `batch_card_picker.html` (opt-in, zero behavior
change for corrections/write-off); a single `Склад:` line in `sale_step_qty_price.html` (via the
existing `_wizard_header.html` partial) and `sale_basket.html`; 3 new regression tests.
</objective>
