<objective>
Fix the receipts wizard's step 2 ("Партия") "Назад" control, which today is a plain
`<a class="button secondary" href="/m/receipts">` full-page link that silently discards the typed
code (verified directly in `receipts_step_batch.html` line 53 during planning — 13-CONTEXT.md's
D-06 called receipts "already correct" but this specific transition was not actually audited at
that granularity). Per the open_question_resolution logged in Plan 13-01, this plan converts it to
the same `hx-get` + fragment pattern used everywhere else in this codebase, closing the last
plain-link exception for receipts. This plan also adds the `?code=` GET query-param pre-fill to
`GET /m/receipts` that Plan 13-05's search quick-action link depends on (D-08) — bundled here
because both changes touch the exact same route function.

Purpose: Phase 13 Success Criterion #2 (uniform hx-get/hx-post Назад pattern, no carve-outs) and
UI-05's route-side prerequisite (a "Принять" quick-action link from search is meaningless until
`/m/receipts` actually consumes `?code=`).
Output: `receipts_step_product.html` (new extracted step-1 fragment), `mobile_pages/receipts.html`
refactored around it, `mobile_receipts.py`'s `mobile_receipt_new` serving step 1 as page-or-fragment
with an optional `code` pre-fill, and `receipts_step_batch.html`'s "Назад" converted to `hx-get`.
</objective>
