<objective>
Close the two remaining gaps `24-VERIFICATION.md` (status: gaps_found, 5/6 truths) and `24-REVIEW.md` (CR-01 critical, WR-03 warning) found after phase 24 shipped:

1. **CR-01 (critical, D-11 violation):** removing the old mobile home tile grid (D-10, correct) deleted the *only* mobile navigation path to `/m/search` (Поиск), `/m/corrections` (Корректировка), and `/m/transfers` (Перемещение). D-11 explicitly names Перемещение as an item that must live on the mobile Товары toolbar mirror — it shipped without it. Поиск and Корректировка have no D-decision authorizing their removal at all (unlike Настройки/Экспорт кассы, which D-12 deliberately and explicitly retires from mobile). All three routes still return 200 by direct URL (proven by `tests/test_mobile_wiring.py::test_every_mobile_tile_path_is_reachable`), but that test proves route registration, not UI reachability.

2. **WR-03 (warning, nav-highlight nit):** `/finance/report` is now reached via Настройки (D-08), but `base.html`'s active-state check still highlights Финансы instead — a misleading "you are here" indicator left over from the plan 24-01 nav-reduction refactor.

**Placement decision for Поиск/Корректировка (Claude's discretion, documented per the gap-closure task's rigor requirement):** Both are added to the SAME mobile Товары toolbar as Перемещение, not to `mobile_pages/home.html`. Reasoning: (a) D-11's own language — "Items that don't fit in the 7 mobile tabs ... are reached mirroring the desktop pattern ... living on the mobile Товары tab. No separate mobile-only navigation scheme for these." — establishes ONE catch-all landing zone, not a scattered set of entry points; splitting Поиск/Корректировка onto Главная while Перемещение lives on Товары would create two different "where do I find the rest" mental models. (b) D-10's entire point was removing Главная's role as a nav-tile hub — re-adding links there would partially reverse that decision. (c) Semantically, Корректировка is a stock-quantity action exactly like Приход/Списание/Перемещение (fits "Действия"), and Поиск is a lookup/reference tool exactly like Справочник/Категории/Каталоги (fits "Справочники") — both slot into the existing two-group shape without inventing a third group or new grouping logic. Note: desktop has never linked to `/corrections` or `/search` from anywhere (confirmed via `git log -p -- app/templates/base.html`, no commit ever added such a link) — this is a pre-existing, out-of-phase-24-scope condition on desktop, not a regression, and this plan does not touch it.

Purpose: Make the phase's own Goal statement — "every secondary action is reachable from the page it belongs to — on desktop and mobile alike" — actually true, and fix the one misleading nav-highlight side effect from the same refactor.
Output: Extended `mobile_partials/products_toolbar.html` (3 new links); corrected `base.html` active-state logic; 3 new regression tests proving rendered-link presence (not just 200-by-URL).
</objective>
