<objective>
Wire the cash ledger into the two existing write paths ŌĆö inside their existing
transactions, at the SERVICE layer (never the route). `register_sale` stages one
aggregated `+total_cents` credit before its trailing commit; `register_return`
flips its `record_operation` from `commit=True` to a staged write, computes the
debit INDEPENDENTLY from the return's own qty ├Ś the origin op's frozen
`unit_price_cents`, and closes stock + cash in ONE `session.commit()`.

Purpose: Implements D-00c (service-layer wiring, commit=False staging, ship
credit + debit together) and D-00d (independent debit computation). This is the
observable behavior of FIN-01 and FIN-02.
Output: sale credit hook, return debit hook + atomic commit, and the integration
tests that prove credit / rollback-writes-zero / debit / partial / atomicity.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sale/return transaction ŌåÆ cash write | The cash credit/debit must live INSIDE the existing stock transaction; a separate commit would let stock and cash diverge on failure. |
| return debit source | The debit must be recomputed from the return's own frozen inputs, never trusted from a prior credit row (which may be legacy/partial). |
| route ŌåÆ service | Wiring must be at the service layer so both desktop and mobile callers credit/debit; a route-level call would silently skip one channel. |
