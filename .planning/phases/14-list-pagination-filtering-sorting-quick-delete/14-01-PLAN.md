<objective>
Build the shared, reusable foundation that every one of the six list pages (products, warehouses, customers, dictionary, catalogs, history) will build on in Wave 2: a page-size constant + page-window/ellipsis helper + Python-side slicer (`app/services/pagination.py`), a copy-paste-ready shared pagination partial (`app/templates/partials/pagination.html`) with its structural CSS, and the `Dictionary.name_lc` Cyrillic-safe shadow column (schema + backfill + write-path maintenance) that Plan 14-03 needs to filter dictionary names correctly.

Purpose: every Wave 2 plan (14-02 through 14-07) depends on this plan and must NOT re-implement page-window math, re-invent the pagination bar markup, or hand-roll a second `_lc` migration pattern. Per RESEARCH.md's "Don't Hand-Roll": six independent implementations of the same "1 2 3 … 8" logic is exactly the kind of duplication that causes an off-by-one bug in exactly one of the six lists.
Output: `app/services/pagination.py` (tested), `app/templates/partials/pagination.html` + 4 new CSS rules in `app/static/style.css`, `Dictionary.name_lc` column live in the schema and populated for all existing + future rows.
</objective>
