<objective>
Give `/customers` its own dedicated filter/sort/page list function — separate from the existing `customer_search_view`/`search_customers` pair, which stay completely UNTOUCHED because `app/routes/sales.py`'s customer picker (`customer_picker.html`, the sale-form autocomplete) depends on them — and retire the now-redundant standalone `/customers/search` route + search box (Pitfall 6), folding equivalent filtering into the main list route's header-row filters.

Purpose: `search_customers` is capped at 20 with a single combined `search_lc` query (name+surname+consultant concatenated) — useful for a quick autocomplete dropdown, but D-04 explicitly wants INDEPENDENT per-column filters (name, surname, consultant number separately) in the table header, which is a genuinely different query shape best kept as its own function rather than overloading the existing search path.
Output: `customers.list_customers_view()` (new); `GET /customers` gains filter/sort/page query params and an is_hx dual response; `GET /customers/search` (route only) is deleted.
</objective>
