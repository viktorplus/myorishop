<objective>
Add year filtering, newest/oldest sorting, and page-number pagination to `/catalogs` â the only one of the six lists that is not a flat table (it groups PDFs by year using a `namespace` tracker that opens/closes a `<table>` per year, currently assuming it owns the WHOLE catalog list in one pass) â by extracting the existing inline grouping loop out of `pages/catalogs.html` into a new `partials/catalog_rows.html` and pagination-slicing the flat, pre-sorted list BEFORE that loop runs (RESEARCH.md Pitfall 5), so the `loop.first`/`loop.last` markers open/close per PAGE instead of per full list.

Purpose: `list_catalogs()` today returns the ENTIRE folder-scanned + membership-joined catalog list with no LIMIT at all; naively slicing that list to 20 rows AFTER the year-grouping template loop has already run (or not slicing before the loop sees the full list) would leave a page with an unclosed `</table>` whenever a page boundary falls mid-year â this plan slices first, then groups, so each page is a structurally complete, self-contained set of `<table>` blocks.
Output: `list_catalogs()` accepts `year`/`sort`/`page` and returns a paginated dict; `/catalogs` renders one swappable `#catalog-rows` block (year select + sort dropdown above the tables, the existing year-grouped tables for just that page's slice, then numbered pagination).
</objective>
