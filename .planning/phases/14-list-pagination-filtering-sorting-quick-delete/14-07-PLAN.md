<objective>
Add year filtering, newest/oldest sorting, and page-number pagination to `/catalogs` — the only one of the six lists that is not a flat table (it groups PDFs by year using a `namespace` tracker that opens/closes a `<table>` per year, currently assuming it owns the WHOLE catalog list in one pass) — by extracting the existing inline grouping loop out of `pages/catalogs.html` into a new `partials/catalog_rows.html` and pagination-slicing the flat, pre-sorted list BEFORE that loop runs (RESEARCH.md Pitfall 5), so the `loop.first`/`loop.last` markers open/close per PAGE instead of per full list.

Purpose: `list_catalogs()` today returns the ENTIRE folder-scanned + membership-joined catalog list with no LIMIT at all; naively slicing that list to 20 rows AFTER the year-grouping template loop has already run (or not slicing before the loop sees the full list) would leave a page with an unclosed `</table>` whenever a page boundary falls mid-year — this plan slices first, then groups, so each page is a structurally complete, self-contained set of `<table>` blocks.
Output: `list_catalogs()` accepts `year`/`sort`/`page` and returns a paginated dict; `/catalogs` renders one swappable `#catalog-rows` block: a synthetic single-row filter/sort header bar (year select + sort select, D-04/Contract B compliant — not a `.filter-bar` div) sitting above the existing year-grouped tables for just that page's slice, then numbered pagination.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-14-21 | Tampering | `sort` query param | mitigate | Resolved through a fixed `!= "oldest"` check — any other value falls back to the default order, never string-interpolated. |
| T-14-22 | Tampering | `year` query param | mitigate | Defensively parsed with `.isdigit()` before `int()` conversion — a non-numeric or malformed value is treated as no filter, never raises a 500. |
| T-14-23 | Tampering (confused deputy) | `page` query param | mitigate | Clamped inside `pagination.paginate()` into `[0, total_pages - 1]`. |
