## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| basket form ŌåÆ register_sale | `code[]`/`qty[]`/`price[]`/`batch_id[]` are untrusted positional arrays; lengths and batch ownership can be forged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-07 | Tampering | parallel `batch_id[]` array | mitigate | structural per-row hidden input + pad to `len(codes)` before zip; missing/short array degrades to ┬½ąÆčŗą▒ąĄčĆąĖčéąĄ ą┐ą░čĆčéąĖčÄ.┬╗, never a positional guess (Pitfall 2) |
| T-09-08 | Tampering | forged/foreign/exhausted `batch_id` | mitigate | `register_sale` rejects a line whose batch is unresolvable or `batch.product_id != product.id`; per-batch oversell aggregates vs current `Batch.quantity` on every POST |
| T-09-09 | Repudiation/Tampering | confirm=1 replay | mitigate | oversell recomputed server-side on every POST; the flag alone is never trusted (existing warn-but-allow contract) |
| T-09-10 | Injection | `row` id in hx attributes | mitigate | reuse `_ROW_ID_RE.fullmatch` validation on `/sales/batch-pick` (routes/sales.py CR-01 precedent) |
| T-09-11 | Tampering (XSS) | batch comment/location in picker | mitigate | Jinja autoescape only; never `|safe` |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | zero packages installed this phase (vacuous gate) |
