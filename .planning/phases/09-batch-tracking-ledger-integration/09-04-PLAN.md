<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| writeoff/correction form → service | `batch_id`, `qty`, `mode`, `value`, `confirm` untrusted; batch may be foreign/exhausted |
| stored batch text → picker HTML | comment/location rendered back into HTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-12 | Tampering | writeoff/correction `batch_id` | mitigate | service resolves the batch and rejects `batch.product_id != product.id` or missing id (LOT-05 enforcement) before any write |
| T-09-13 | Tampering / integrity | correction count baseline | mitigate | diff against `batch.quantity` not `product.quantity` (Pitfall 7) so a recount cannot corrupt sibling batches |
| T-09-14 | Repudiation/Tampering | correction/writeoff confirm=1 replay | mitigate | over-removal recomputed server-side vs current `Batch.quantity` on every POST; flag never trusted alone |
| T-09-15 | Tampering (XSS) | batch text in picker | mitigate | Jinja autoescape only; never `|safe` |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | zero packages installed this phase (vacuous gate) |
</threat_model>
