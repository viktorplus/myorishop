<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /transfers routes | Form/query fields (`code`, `qty`, `batch_id`, `dest_warehouse_id`, `confirm`) are untrusted; a stale form can post a deleted/same warehouse or a foreign batch |
| stored batch/product text → transfer HTML | product name, batch comment/location rendered back into the picker/recent list |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-06 | Tampering | client `batch_id` on lookup/batch-pick/POST | mitigate | batch-pick re-validates `candidate.product_id == product.id` before echoing; register_transfer (Plan 01) re-checks ownership before any write |
| T-10-07 | Tampering | destination `<select>` posting the source or an inactive warehouse | mitigate | dest select is server-filtered to active minus source; register_transfer re-rejects `dest == source.warehouse_id` and `dest ∉ active` server-side (client-side filtering is defense-in-depth only) |
| T-10-08 | Tampering (XSS) | product name / batch comment / location in the picker + recent list | mitigate | Jinja autoescape only, never `|safe` (batch_picker.html precedent) |
| T-10-09 | DoS / integrity | commit-time failure leaving the session pending-rollback | mitigate | POST wraps register_transfer in try/except → session.rollback() + log + 422 form partial (Pitfall 7) |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | zero external packages installed this phase (vacuous gate) |
</threat_model>
