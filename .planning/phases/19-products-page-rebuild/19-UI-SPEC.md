## Interaction & Layout Decisions

These are not covered by the fixed template sections above but are load-bearing for the
executor; each traces to a specific `19-RESEARCH.md` finding.

1. **Batch expand default state: collapsed.** Render `<details>` without the `open` attribute.
   Rationale (RESEARCH Assumption A1): a fully-expanded 20-row page with every product's batches
   shown would defeat the phase goal ("reads as a stock list the operator can scan by code, not
   a flat per-batch dump"). This is an assumption made in the absence of CONTEXT.md — flag to the
   operator during UAT if they expected always-visible batches.
2. **Quantity column position:** insert directly after "Категория" and before "Закупочная"
   (ДЦ)/"Продажа" (ПЦ), right-aligned with the existing `.num` class (tabular-nums). Table
   `colspan` for the new batch sub-row increases from 6 to 7 accordingly.
3. **Category filter: no redesign.** Keep the existing free-text `<input name="category">`
   unchanged — PROD-08 is already satisfied by it (tested, working). Optional non-blocking
   polish, planner's discretion: add `list="cat-options"` sourced from the existing
   `category_options()` helper, mirroring `product_form.html:54-56` — not required for any
   success criterion.
4. **Delete link accessibility (PROD-02, WCAG 1.4.1):** `<a href="#" class="link-danger" hx-post="/products/{{ product.id }}/quick-delete?..." hx-confirm="..." hx-target="#product-rows" hx-swap="outerHTML">Удалить</a>`.
   Keep the browser's default underline (do **not** set `text-decoration: none`) so the link
   reads as interactive by shape, not color alone. `href="#"` is mandatory, never a real delete
   URL — htmx intercepts the click via `hx-post`; a real `href` would let a no-JS/slow-first-paint
   user GET a POST-only route (405).
   New CSS (from RESEARCH Pattern 3, additive only, no new color role):
   ```css
   a.link-danger { color: #b91c1c; }
   a.link-danger:hover { color: #7f1414; }
   ```
5. **Batch sub-row must contain zero `<input>`/`<select>` elements** — the existing
   filter/sort/pagination state machine sweeps up *every* input/select inside `#product-rows`
   via `hx-include`. The batch `<details>` block is read-only (RESEARCH Pitfall 1 guard).
6. **Legacy/NULL batch fields render as "—", never blank** — guard every batch field exactly
   like `batch_picker.html:52`: `{% if b.expiry %}{{ b.expiry|ru_date }}{% else %}<span class="muted">—</span>{% endif %}` (RESEARCH Pitfall 4).
7. **Zero-batch products render no expand affordance at all** — the `<details>` sub-row is
   conditional on `{% if product_batches %}` (RESEARCH Pitfall 5); no empty/broken disclosure
   triangle for products with no open batches (e.g. brand-new products, or fully-depleted stock).
