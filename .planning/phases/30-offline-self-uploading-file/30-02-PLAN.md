<objective>
Lay the shared contracts every later Phase-30 wave consumes: (1) the wire integrity
field `payload_sha256` on the NDJSON header (OFF-07 emit side), (2) one public
`collect_push_records` so the offline export and the online push share ONE unsynced
collector (SYNC-04), (3) the `/api/offline/` auth-guard bypass so the token-guarded
ingest routes are reachable — in-body auth, no CSRF/cookie machinery (D-05), exact
prefix (Pitfall 3), and (4) a pure `app/services/offline.py` minting/verifying the
short-lived upload-scoped token (D-03) and gating the schema version (D-09). All four
are small additive edits to existing, tested, both-dialect-proven code — no new
package, no migration.

Purpose: interface-first. Ingest (30-03) and export (30-04) receive stable
contracts (digest, collector, token, schema gate, bypass) instead of rediscovering them.

Output: edited `merge.py` / `sync_client.py` / `security.py` + new `app/services/offline.py`.
</objective>
