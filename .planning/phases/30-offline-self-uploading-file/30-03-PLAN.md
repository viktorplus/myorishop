<objective>
Build the server ingest: `app/routes/offline.py` with the two-step handshake
`POST /api/offline/login` (creds → short-lived upload token, D-03/D-04) and
`POST /api/offline/upload` (token + NDJSON → integrity + schema gate → same
`apply_merge` in one owned transaction, D-08/D-09/OFF-05), plus the S2 RU result
pages the operator lands on (D-01). The upload route is a near-verbatim mirror of
`sync_push`; the ONLY additions are the SHA-256 digest check and the schema-version
gate, both at the route layer BEFORE `apply_merge` — never inside pure `parse_exchange`.

Purpose: OFF-04 (auth, no data on failure — D-04), OFF-05 (idempotent, all-or-nothing),
OFF-07 (integrity + schema validation with distinct RU messages — D-08/D-09).

Output: `app/routes/offline.py`, `app/templates/offline/result.html`, `main.py` wiring.
</objective>

    `request: Request`, `token: str = Form(...)`, `payload: str = Form(...)` (a form FIELD —
    CRLF-normalized on submit, Pitfall 1), `session: Session = Depends(get_session)`. Gates IN
    THIS ORDER, all BEFORE any DB write:
    1. Token: `try: offline_service.verify_offline_token(token)` except
       `(SignatureExpired, BadSignature)` → `return _result(request, "expired", status=401)`.
    2. Size cap: if `len(payload.encode("utf-8")) > MAX_OFFLINE_BYTES` →
       `return _result(request, "corrupted", status=413)` (belt-and-braces DoS cap, mirror
       sync.py MAX_PUSH_BYTES, independent of any proxy).
    3. Canonicalize: `lines = payload.splitlines()` (strips CRLF and LF alike → identical bytes
       regardless of transport normalization). If not lines → `_result(request, "corrupted",
       status=400)`.
    4. Integrity (D-08): `header = json.loads(lines[0])`; `record_lines = lines[1:]`; if
       `payload_digest(record_lines) != header.get("payload_sha256")` →
       `return _result(request, "corrupted", status=400)` («Файл повреждён…»). Wrap the
       `json.loads` in try/except → corrupted on failure (never echo the bytes).
    5. Schema gate (D-09): `server_schema = current_schema_version(session)`; if NOT
       `offline_service.schema_version_ok(header.get("schema_version", ""), server_schema)` →
       `return _result(request, "incompatible", status=409, file_ver=header.get("schema_version"),
       server_ver=server_schema)` (both versions named; empty server schema skips per 30-02).
    6. Structural: `try: batch = parse_exchange(lines)` except `ValueError` →
       `return _result(request, "corrupted", status=400)` (this enforces `format_version`).
    7. All-or-nothing merge (OFF-05, copy sync_push verbatim): `session.rollback()`; `with
       session.begin(): report = apply_merge(session, batch, server_now=utcnow_iso())`. On
       success `return _result(request, "success", total=report.operations_inserted +
       report.cash_inserted + sum(report.reference_inserted.values()), ops=report.operations_inserted,
       cash=report.cash_inserted)` — note `report.reference_inserted` is a `dict[str, int]`
       (per-kind counts), so it MUST be summed via `.values()`, never added as a bare dict.
       A poisoned record (e.g. missing FK parent) rolls the whole
       batch back — the all-or-nothing guarantee (do NOT swallow the IntegrityError; the tests
       assert the rollback left zero rows, mirroring `test_push_all_or_nothing`).
