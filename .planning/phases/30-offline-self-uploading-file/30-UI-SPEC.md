# Phase 30 — UI Design Contract

> Visual and interaction contract for the offline self-uploading file surface. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** this is a Python/FastAPI/Jinja2/HTMX/Pico-less-plain-CSS app — NOT React. shadcn/registry gates are **not applicable**. The project already has a locked design system (Phase 2 UI-SPEC, encoded in `app/static/style.css`); this phase **reuses it verbatim** and adds **zero new tokens**. The only novel constraint is that one surface (the standalone `.html` file) cannot link the server stylesheet and must **inline-replicate** the existing tokens.

---

## Surfaces In Scope

This phase's UI is deliberately minimal and split across two runtimes. Exactly three surfaces:

| # | Surface | Runtime | Stylesheet available? |
|---|---------|---------|-----------------------|
| S1 | **Self-uploading HTML file** (login form + client-side preview + confirm + form-POST) | Any browser on an internet PC, `file://` origin, **no app install** | ❌ No — must **inline-replicate** the design tokens (self-contained, no CDN/font) |
| S2 | **Server result pages** (success / wrong-password / corrupted-file / incompatible-version) the operator lands on after the upload navigation | Central server (Jinja2) | ✅ Yes — reuses `/static/style.css` via a minimal no-session shell |
| S3 | **Read-only "offline-exported, not yet confirmed" hint** on the desktop client + the **export trigger** on the client export page | Desktop client (Jinja2, existing `base.html`) | ✅ Yes — reuses `/static/style.css` |

**Out of scope (do not redesign):** the app-wide nav, the online-sync header partial, any mobile UI, the merge engine internals, any PULL surface. S3's hint is **OPTIONAL** (Claude's Discretion, D-07) — the planner MAY implement it; it must **never** stamp `synced_at`.

## Copywriting Contract

All copy is Russian (project UI language). Voice matches the existing app: `h1` title, `.muted` captions, `.error-block` for rejections, verb-first buttons ("Войти", "Скачать …"). Generic auth failure — **no user-enumeration** (mirror the existing `BAD_CREDENTIALS_ERROR` constant in `app/services/auth.py`).

|-------|--------------|------|

| Expired/invalid upload token (belt-and-braces) | `Сессия истекла` | `.error-block`: `Время на загрузку истекло. Войдите заново и повторите отправку.` |

### S3 — Client export trigger + optional hint

| Element | Copy |
|---------|------|
| Export CTA (`a.button`, primary — session-guarded client page) | `Экспортировать офлайн-файл` |
| Export helper (`.muted`) | `Соберёт всё, что ещё не отправлено на сервер, в один файл для USB-флешки. Файл откроется в браузере на компьютере с интернетом и загрузит себя сам.` |
| Export blocked — server URL not set (Pitfall 5) | `.error-block`: `Сначала укажите адрес сервера в настройках, затем экспортируйте файл.` |

**Destructive actions:** none. The upload is additive/idempotent (insert-if-new); there is no delete, overwrite, or irreversible action in this phase, so no destructive-confirmation copy is required. The **confirm** step (OFF-06) is a *preview-then-send* gate, not a destructive confirmation — styled with the accent primary button, not `button.danger`.

## Surface Contracts (Interaction & Layout)

### S1 — Self-uploading file (the one genuinely new surface)

**Hard self-containment rules (OFF-03):**
- **Inline `<style>` only.** No `<link rel="stylesheet">`, no `@import`, no web-font `@font-face`, no CDN, no external image. The internet PC may be firewalled to everything but the server — assume zero external asset fetches succeed.
- **Inline-replicate the exact tokens** below (the file cannot reach `/static/style.css`). These MUST match `style.css` values so the file looks like the app:

  | Selector | Declarations (verbatim tokens) |
  |----------|--------------------------------|
  | `body` | `font-family: system-ui, -apple-system, "Segoe UI", sans-serif; margin:0; background:#fafafa; color:#222; font-size:16px; line-height:1.5;` |
  | container (`main`) | `max-width:480px; margin:0 auto; padding:16px 16px 32px;` (mirrors `.mobile-shell` / auth width) |
  | `h1` | `font-size:24px; font-weight:600; line-height:1.2;` |
  | `h2` | `font-size:20px; font-weight:600; line-height:1.2;` |
  | `label` | `display:block; font-size:14px; margin-bottom:4px;` |
  | `.field` | `margin-bottom:16px;` |
  | `input` | `padding:8px; border:1px solid #d9d9d9; border-radius:4px; font-size:16px; width:100%; box-sizing:border-box;` |
  | `button` (primary) | `padding:8px 16px; background:#2563eb; color:#fff; border:1px solid #2563eb; border-radius:4px; font-size:16px; cursor:pointer;` |
  | `button:disabled` | `opacity:0.6; cursor:not-allowed;` |
  | focus | `:focus-visible { outline:2px solid #2563eb; outline-offset:1px; }` |
  | `.muted` | `color:#6b7280; font-size:14px;` |
  | `.error-block` | `background:#fdecec; color:#b91c1c; padding:8px 16px; border-radius:4px; margin-bottom:16px;` |
  | preview panel | `background:#fff; border:1px solid #d9d9d9; border-radius:4px; padding:16px;` (mirrors `.metric-tile`/`.customer-chip` surface) |

- **`lang="ru"`, `<meta charset="utf-8">`, `<meta name="viewport" width=device-width>`** — mirror `base.html`/`auth_base.html` head.

**Interaction flow (locked by D-01/D-02/D-04/D-10):**
1. On load, JS reads the embedded `<script type="application/x-ndjson">` `textContent`, reverses the `<\/script>` escape, parses the header line, and renders the **preview counts** — **before any network call**.
2. Operator types login + password, clicks **"Проверить"** → `fetch` POST `application/x-www-form-urlencoded` (creds only) to `/api/offline/login`. On failure: show the inline `.error-block` message; **the confirm step stays hidden and no payload is ever transmitted** (OFF-04). On success: stash the returned token in a hidden field and **reveal** the confirm step.
3. The **"Отправить на сервер"** confirm button is the ONLY control that triggers the top-level `<form method="post" enctype="multipart/form-data" action="{server}/api/offline/upload">` navigation. Nothing uploads until this click (OFF-06).
4. The navigation lands the operator on an S2 result page.

**Visual states S1 must render:** (a) preview-with-counts (default), (b) preview-empty ("нет данных"), (c) creds-entry, (d) login-error (three variants above), (e) authenticated / confirm-ready (confirm button revealed), (f) submitting (button `disabled`, no spinner graphic — the project convention is `button:disabled { opacity:0.6 }`, never an icon/animation).

**Loading indicator:** reuse the project's no-graphic convention — disable the button and/or show a `.muted` "Отправка…" caption. **No spinner, no icon, no animation asset** (consistent with the header sync indicator, which is muted text only).

### S2 — Server result pages

- Render through a **minimal no-session shell** (mirror `auth_base.html`: `mobile-shell` container, `/static/style.css`, **no `<nav>`**, no `current_user`). The offline operator has **no cookie/session** on the server, so `base.html`'s nav/sync chrome must not be used.
- Each state = one `h1` + one body block (`.muted` for success, `.error-block` for the three rejections) using the exact copy above. Single-column, `max-width:480px`, centered.
- **Never** interpolate raw uploaded bytes into the page except the two known version strings in the incompatible message (autoescaped). No merge-report dump, no stack trace, no echoed filename (mirrors `sync.py` fixed-constant errors, T-28-07).

### S3 — Client export trigger + optional hint

- The **export CTA** lives on a session-guarded client page (existing `pages/export.html` is the natural host, or a small dedicated card) as `<a class="button" href="/offline/export">Экспортировать офлайн-файл</a>` with the `.muted` helper beneath — mirrors the existing CSV-export button idiom verbatim.
- If `settings.sync_server_url` is empty, replace the button with the `.error-block` "укажите адрес сервера" message (Pitfall 5).
- The **optional hint** (D-07 discretion), if implemented, is a plain `.muted` caption derived from a lightweight local export-timestamp marker (a setting/file) — **it must never read from or write to `synced_at`**, and must **not** reuse the amber badge geometry (that is the live online unsynced count). Keep it visually distinct as muted body text.

## Accessibility & Offline Constraints

- **Contrast:** all token pairs already meet WCAG AA in the shipped app (`#222` on `#fafafa`, `#fff` on `#2563eb`, `#b91c1c` on `#fdecec`). No new pair introduced.
- **Focus:** the `2px solid #2563eb` outline must be preserved in S1's inline CSS (never `outline:none` without replacement — project rule).
- **Keyboard:** login form is native `<form>`/`<input>`/`<button>`; confirm is a native button — all keyboard-operable by default.
- **Colour-independence:** errors carry text ("Данные не отправлены", "Файл повреждён…"), never colour alone.
- **No motion:** no animation/spinner assets anywhere (offline-asset constraint + project convention).
