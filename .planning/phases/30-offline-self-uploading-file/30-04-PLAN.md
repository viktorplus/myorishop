<objective>
Ship the offline client half: `GET /offline/export` (session-guarded desktop route)
collects the unsynced ledger + its FK closure via the shared `collect_push_records`,
serializes to NDJSON, and renders a SINGLE self-contained HTML file
(`self_upload.html`) that the operator saves to a USB drive. Opened in any browser on
an internet PC with no app install, the file shows a client-side preview of counts,
authenticates through the two-step handshake, and — only after an explicit confirm —
posts itself to `/api/offline/upload`. Plus the S3 export CTA on the existing export page.

Purpose: OFF-01 (export reads accumulated unsynced work, never stamps synced_at),
OFF-02 (single self-contained USB file, offline), OFF-03 (no install; opens in a
browser; uploads itself), OFF-06 (preview counts + explicit confirm).

Output: `offline_export` route in `app/routes/offline.py`, new
`app/templates/offline/self_upload.html`, edited `app/templates/pages/export.html`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| desktop operator → /offline/export | session-guarded; only a logged-in operator can produce the file |
| exported file (embedded data) → any browser | the embedded NDJSON is non-executable; the only breakout vector `</script>` is escaped at render |
| file → server URL | the absolute server URL is embedded at export time from settings.sync_server_url; a blank URL refuses to export |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-08 | Tampering (HTML breakout) | self_upload.html embed | mitigate | `<script type="application/x-ndjson">` (non-executable) + `</script>`→`<\/script>` escape at render + `| safe` only after the escape; reversed in JS before parse/submit; round-trip proven (test_script_tag_escaping_round_trip) |
| T-30-06 | Access Control | /offline/export | mitigate | export route stays session-guarded (NOT under /api/offline/); anonymous → 303 /login (test_offline_bypass_is_narrow export direction) |
| T-30-01 | Tampering | digest over exported body | mitigate | the exporter digests LF-joined record lines (via serialize_exchange/payload_digest from 30-02); the JS submits the ORIGINAL NDJSON so the server's canonicalized recompute matches |
| T-30-03 | Info Disclosure | self_upload.html flow | mitigate | two-step handshake in JS: creds fetched first; the payload form is submitted only after a confirmed login and an explicit confirm click (OFF-04/OFF-06) |
| — (data-loss avoidance) | — | offline_export | accept | D-07: export only reads, never stamps synced_at — a lost/never-uploaded file keeps its rows visible; consequence: a permanently-offline client's unsynced badge stays inflated (flagged for UAT) |
| T-30-SC | Tampering | package installs | accept | Zero new packages; the file uses baseline browser fetch/form/textContent (no polyfill, no CDN) |
