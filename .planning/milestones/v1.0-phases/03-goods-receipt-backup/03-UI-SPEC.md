## Spacing Scale

Declared values (must be multiples of 4). Identical to Phase 2 — already implemented in `style.css`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between label and input; error text top margin |
| sm | 8px | Gaps inside form rows; table cell padding (8px 12px); button vertical padding; nav padding |
| md | 16px | Default gap between form fields (`.field`); button horizontal padding; error-block padding |
| lg | 24px | Gap between page sections (receipt form → «Последние приходы»; backup actions → backup table → restore instructions) — `.form-actions` top margin |
| xl | 32px | Page top padding under nav (existing `.container`/nav margins) |
| 2xl | 48px | Empty-state block vertical padding (`.empty-state`) |
| 3xl | 64px | not used this phase |

Exceptions: 12px — dense table cell horizontal padding only (existing `th, td { padding: 8px 12px }`; multiple of 4, carried from Phase 2). Control heights derived from scale: inputs/buttons 36px (8px vertical padding + 16px text + borders).

## Typography

Exactly 4 sizes, 2 weights — identical to Phase 2. No other values permitted.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (inputs, table cells, paragraphs, restore instructions) | 16px | 400 | 1.5 |
| Label (form labels, table headers, muted meta: dates/sizes/hints) | 14px | 400 (labels/hints) / 600 (table `th`) | 1.5 |
| Heading (h2 section titles: «Последние приходы», «Восстановление из копии») | 20px | 600 | 1.2 |
| Display (h1 page titles: «Приход товара», «Резервные копии») | 24px | 600 | 1.2 |

- Weights: 400 (regular) and 600 (semibold) only.
- Numeric columns (quantity, prices, file size): 16px, 400, right-aligned, `font-variant-numeric: tabular-nums` (existing `.num` class).
- No italics, no uppercase-tracking treatments.

## Color

Identical to Phase 2 — no new colors introduced this phase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` | Page background (existing), text `#222` |
| Secondary (30%) | `#ffffff` | Tables, form surfaces, nav bar; borders `#d9d9d9`; table header background `#f0f0f0` |
| Accent (10%) | `#2563eb` | See reserved list below; hover/active shade `#1d4ed8` |
| Destructive | `#b91c1c` | Inline validation error text and error blocks only (background `#fdecec`) — this phase has NO destructive buttons |

Accent reserved for: primary submit buttons («Сохранить приход», «Создать резервную копию»), text links (nav links; active nav link = accent + 600 weight), input `:focus` outline (2px, existing rule). Nothing else. No secondary buttons needed this phase; if one appears, it uses the neutral `button.secondary` style (white, `#d9d9d9` border, `#222` text).

Muted text (`.muted`, `#6b7280`): timestamps in recent receipts, backup file dates/sizes, field hints, «(необязательно)» markers, empty-state body text. Tint of the dominant neutral, not a fifth color role.

Success feedback: neutral — plain `#222` text on the page surface (see copy «Приход сохранён…»), no green. Semantic color budget stays at exactly one (destructive), per Phase 2 convention.

## Interaction Contract

- **Save-and-next loop (D-02, core value — fast entry):** `hx-post="/receipts"` targeting the form partial. Success → 200 with a fresh empty form + success line + `hx-swap-oob` recent-receipts rows; validation failure → 422 with the form re-rendered, entered values preserved, inline errors (422 swap already whitelisted in base.html htmx-config). Focus: explicit `hx-on::htmx:load` on the fresh form partial calling `document.getElementById('code').focus()` — do NOT rely on `autofocus` inside swapped content (RESEARCH Pitfall 6). Keep `autofocus` on the code input for initial page load.
- **Code lookup (D-03 / RCP-02):** `hx-get="/receipts/lookup"` on the code input, `hx-trigger="input changed delay:300ms"`, `hx-sync="this:replace"`, `hx-include` guarded fields. Server decides: product match → fragment filling name + three price inputs (single wrapper fragment, one `hx-target`); dictionary-only match → name; nothing / fields already typed → 204 no swap. Swap-time guard (`hx-on::htmx:before-swap` → `shouldSwap = false` when operator-typed values present) replicated from Phase 2 `product_form.html` — lookup must never clobber typed input (Pitfall 7).
- **Keyboard-first:** form submits on Enter from any field; tab order = visual order (Код → Название → Количество → prices → submit); visible focus state (existing 2px accent outline) on all interactive elements.
- **Buttons during request:** `hx-disabled-elt="this"` on submit buttons; existing `button:disabled { opacity: .6 }`.
- **Money inputs:** `type="text" inputmode="decimal"`, comma and dot accepted (server `to_cents`); quantity `type="text" inputmode="numeric"`, positive integer only. Display money via `format_cents`, comma decimal separator.
- **Backup now:** `hx-post="/backup"` re-renders the backup list + success line; button disabled during request (VACUUM can take a moment). Failure → error block (copy above), list unchanged.
- **No client-side JS beyond vendored HTMX.** No modals, no toasts; htmx 2.x `hx-on::` double-colon syntax only.
- **Manual UAT gates (from RESEARCH validation map):** after save, focus lands on «Код» without a click; lookup never overwrites typed values; restore.bat verified once end-to-end.
