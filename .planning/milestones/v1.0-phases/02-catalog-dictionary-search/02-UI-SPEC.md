## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between label and input; inline padding tweaks |
| sm | 8px | Gaps inside form rows; table cell padding (8px 12px); button padding vertical |
| md | 16px | Default gap between form fields; padding inside cards/panels; button padding horizontal |
| lg | 24px | Gap between page sections (search block → results; form → history table) |
| xl | 32px | Page top padding under header/nav |
| 2xl | 48px | Empty-state block vertical padding |
| 3xl | 64px | not used this phase |

Exceptions: 12px — dense table cell horizontal padding only (justified: multiple of 4, keeps 20-row tables compact). Control heights derived from the scale: inputs and buttons 36px tall (8px vertical padding + 16px text + borders).

**Existing CSS uses rem values off-scale (0.4rem, 0.6rem, 0.9rem = 6.4/9.6/14.4px).** This phase normalizes them to the scale above when touching those rules (table cells → `8px 12px`, buttons → `8px 16px`, inputs → `8px`). 12px is permitted as a multiple-of-4 value for dense table cell horizontal padding.

## Typography

Exactly 4 sizes, 2 weights. No other values permitted in this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (inputs, table cells, paragraphs) | 16px | 400 | 1.5 |
| Label (form labels, table headers, meta text like dates/author, hints) | 14px | 400 (labels/hints) / 600 (table `th`) | 1.5 |
| Heading (h2 section titles: «История цен», form section titles) | 20px | 600 | 1.2 |
| Display (h1 page title: «Товары», «Справочник», «Новый товар») | 24px | 600 | 1.2 |

- Weights: 400 (regular) and 600 (semibold) only. Existing `h1 { font-size: 1.5rem }` = 24px — keep, set `font-weight: 600`.
- Numeric columns (prices, quantities): body 16px, 400, right-aligned, `font-variant-numeric: tabular-nums`.
- No italics, no uppercase-tracking treatments this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` | Page background (existing), text on it `#222` (existing) |
| Secondary (30%) | `#ffffff` | Table/card/form panel surfaces, nav bar; borders `#d9d9d9`; table header background `#f0f0f0` (existing) |
| Accent (10%) | `#2563eb` | See reserved list below; hover/active shade `#1d4ed8` |
| Destructive | `#b91c1c` | «Удалить товар» button/link, inline validation error text, error message blocks (error block background `#fdecec`) |

Accent reserved for: primary submit buttons («Сохранить товар», «Сохранить код», «Добавить товар», «Добавить код»), text links (including nav links; active nav link = accent + 600 weight), input `:focus` border/outline, and the matched-substring highlight in search results (`<mark>` styled as accent-tinted background `#e8effd`, text unchanged `#222`). Nothing else. Secondary buttons («Отмена», «Восстановить») are neutral: white background, `#d9d9d9` border, `#222` text.

Muted text (timestamps, «кто изменил», hints, empty-state body): `#6b7280`. This is a tint of the dominant neutral, not a fifth color role.

Success feedback (after save): neutral — plain text message `#222` on secondary surface («Сохранено»), no green; keeps semantic color budget at exactly one (destructive).

| Error state (form validation) | Inline under the field, destructive color. Examples: «Укажите код товара.», «Укажите название.», «Код уже используется другим товаром — введите другой код.», «Неверный формат цены — введите число, например 12,50.» |

Search input placeholder: «Код или название товара…» (autofocused on /products).

Focal point per screen (explicit): /products — search input; /products/new and /products/{id}/edit — first field «Код»; /dictionary — inline add row (code input).

## Interaction Contract

- **Instant search:** `hx-get` on input, `hx-trigger="input changed delay:300ms, keyup[key=='Enter']"`, target = results table body partial. Debounce 300ms is a locked decision. No loading spinner needed at local latency; `hx-indicator` optional.
- **Dictionary autofill:** `hx-get` from the code input, debounce 300ms; response fills name field only when name is empty; show autofill hint (see copy).
- **Keyboard-first (core value: fast operator entry):** every form submits on Enter; logical tab order = visual order; search input autofocus on /products; first field autofocus on forms; visible focus state (accent outline, 2px) on all interactive elements — never `outline: none` without replacement.
- **Buttons during request:** existing `button:disabled { opacity: .6 }` pattern retained; disable via `hx-disabled-elt="this"`.
- **Money inputs:** `type="text" inputmode="decimal"`, accept comma and dot (server-side `to_cents` already handles); display prices via `format_cents` with comma decimal separator.
- **No client-side JS beyond vendored HTMX.** No modals, no toasts libraries; confirmations via `hx-confirm`.
