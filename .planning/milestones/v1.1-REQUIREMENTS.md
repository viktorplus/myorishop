# Requirements Archive: v1.1 Multi-Warehouse & Batch Tracking

**Status:** ✅ SHIPPED 2026-07-13
**Defined:** 2026-07-10
**Core Value:** The operator can quickly and reliably record receipts and sales so stock counts and profit figures are always correct — without losing any data.

## v1.1 Requirements (all complete)

### Warehouses & Locations

- [x] **WH-01**: User can create and manage multiple warehouses — Phase 8
- [x] **WH-02**: Stock item has an optional free-text storage location tag within its warehouse — Phase 9
- [x] **WH-03**: User can transfer stock (a batch or part of it) from one warehouse to another without losing cost/price history — Phase 10

### Categories

- [x] **CAT-01**: "Товары на складе" page groups products by category/rubric — Phase 7

### Batches & Lots

- [x] **LOT-01**: A product code can have multiple batches (lots), each with its own expiry date and price — Phase 9
- [x] **LOT-02**: At sale, operator sees a list of matching batches (price, expiry, remaining quantity, comment) and manually selects one — Phase 9
- [x] **LOT-03**: Optional expiry date field per batch — Phase 9
- [x] **LOT-04**: Optional free-text comment per batch, shown in the sale-time batch picker — Phase 9
- [x] **LOT-05**: Write-off, return, and stock correction also require selecting the specific batch, not just the product — Phase 9
- [x] **LOT-06**: Report of batches with an approaching or passed expiry date — Phase 10

### Pricing Guardrails

- [x] **PRICE-01**: Optional minimum sale price per product — selling below it shows a warning but allows override — Phase 7

### UI

- [x] **UI-01**: A dedicated mobile flow — simpler, single-purpose screens/steps for core operations — rather than adapting the same dense desktop pages via CSS alone — Phase 11 (completed 2026-07-13, UAT 6/6 passed, security-verified)

## v2 Requirements (deferred, unchanged)

Tracked but not in this milestone's scope — carried forward to v2.0 planning.

### Sync & Multi-User

- **SYNC-V2-01**: Multi-operator sync across countries via a central server (server-based + USB flash-drive sync)
- **CUR-V2-01**: Multi-currency support
- **AUTH-V2-01**: User roles: administrator, operator, report viewer

### Customer Intelligence

- **CST-V2-01**: Customer purchase-frequency analysis and "running low" reminders
- **CST-V2-02**: On goods receipt, show customers likely interested in the product

### Export

- **EXP-V2-01**: CSV export includes warehouse/batch columns
